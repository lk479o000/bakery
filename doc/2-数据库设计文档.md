# 数据库设计文档

## 1. 用户模块

### 1.1 t_user（用户表）
存储小程序用户信息

| 字段名 | 类型 | 长度 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | VARCHAR | 32 | 否 | - | 主键，用户唯一标识 |
| openid | VARCHAR | 64 | 否 | - | 微信openid，唯一 |
| nickname | VARCHAR | 64 | 是 | NULL | 用户昵称 |
| avatar | VARCHAR | 255 | 是 | NULL | 用户头像URL |
| phone | VARCHAR | 20 | 是 | NULL | 手机号 |
| points | INT | - | 否 | 0 | 积分余额 |
| balance | DECIMAL | 10,2 | 否 | 0.00 | 账户余额（元） |
| member_level | TINYINT | - | 否 | 1 | 会员等级：1-普通会员 |
| created_at | TIMESTAMP | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | - | 否 | CURRENT_TIMESTAMP | 更新时间 |

```sql
CREATE TABLE t_user (
    id VARCHAR(32) PRIMARY KEY COMMENT '主键，用户唯一标识',
    openid VARCHAR(64) UNIQUE NOT NULL COMMENT '微信openid',
    nickname VARCHAR(64) COMMENT '用户昵称',
    avatar VARCHAR(255) COMMENT '用户头像URL',
    phone VARCHAR(20) COMMENT '手机号',
    points INT DEFAULT 0 COMMENT '积分余额',
    balance DECIMAL(10,2) DEFAULT 0.00 COMMENT '账户余额（元）',
    member_level TINYINT DEFAULT 1 COMMENT '会员等级：1-普通会员',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) COMMENT='用户表';
```

## 3. 订单模块

### 3.1 t_order（订单表）
存储订单信息

| 字段名 | 类型 | 长度 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | VARCHAR | 32 | 否 | - | 主键，订单唯一标识 |
| user_id | VARCHAR | 32 | 否 | - | 用户ID，关联t_user.id |
| order_no | VARCHAR | 32 | 否 | - | 订单编号，唯一 |
| type | ENUM | - | 否 | 'store' | 订单类型：store-门店订单，recharge-储值订单，payment-买单订单，coupon-券包订单 |
| status | TINYINT | - | 否 | 0 | 订单状态：0-待支付，1-已支付，2-制作中，3-待取餐，4-已完成，5-已取消 |
| total_amount | DECIMAL | 10,2 | 否 | - | 订单总金额（元） |
| discount_amount | DECIMAL | 10,2 | 否 | 0.00 | 优惠金额（元） |
| pay_amount | DECIMAL | 10,2 | 否 | - | 实付金额（元） |
| pay_type | ENUM | - | 是 | NULL | 支付方式：wechat-微信支付，balance-余额支付 |
| remark | VARCHAR | 255 | 是 | NULL | 订单备注 |
| create_time | TIMESTAMP | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| pay_time | TIMESTAMP | - | 是 | NULL | 支付时间 |

```sql
CREATE TABLE t_order (
    id VARCHAR(32) PRIMARY KEY COMMENT '主键，订单唯一标识',
    user_id VARCHAR(32) NOT NULL COMMENT '用户ID，关联t_user.id',
    order_no VARCHAR(32) UNIQUE NOT NULL COMMENT '订单编号',
    type ENUM('store','recharge','payment','coupon') DEFAULT 'store' COMMENT '订单类型：store-门店订单，recharge-储值订单，payment-买单订单，coupon-券包订单',
    status TINYINT DEFAULT 0 COMMENT '订单状态：0-待支付，1-已支付，2-制作中，3-待取餐，4-已完成，5-已取消',
    total_amount DECIMAL(10,2) NOT NULL COMMENT '订单总金额（元）',
    discount_amount DECIMAL(10,2) DEFAULT 0.00 COMMENT '优惠金额（元）',
    pay_amount DECIMAL(10,2) NOT NULL COMMENT '实付金额（元）',
    pay_type ENUM('wechat','balance') COMMENT '支付方式：wechat-微信支付，balance-余额支付',
    remark VARCHAR(255) COMMENT '订单备注',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    pay_time TIMESTAMP NULL COMMENT '支付时间',
    INDEX idx_user_id (user_id) COMMENT '用户ID索引',
    INDEX idx_order_no (order_no) COMMENT '订单编号索引'
) COMMENT='订单表';
```

### 4.1 t_coupon（优惠券表）
存储优惠券模板信息

| 字段名 | 类型 | 长度 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | VARCHAR | 32 | 否 | - | 主键，优惠券唯一标识 |
| name | VARCHAR | 64 | 否 | - | 优惠券名称 |
| type | ENUM | - | 否 | - | 优惠券类型：full_reduction-满减券，discount-折扣券 |
| value | DECIMAL | 10,2 | 否 | - | 优惠值：满减券为减免金额，折扣券为折扣率（如0.8表示8折） |
| min_amount | DECIMAL | 10,2 | 否 | 0.00 | 最低消费金额（元） |
| start_time | TIMESTAMP | - | 否 | - | 开始时间 |
| end_time | TIMESTAMP | - | 否 | - | 结束时间 |
| total_count | INT | - | 否 | 0 | 发放总量 |
| received_count | INT | - | 否 | 0 | 已领取数量 |
| status | TINYINT | - | 否 | 1 | 状态：0-禁用，1-启用 |

```sql
CREATE TABLE t_coupon (
    id VARCHAR(32) PRIMARY KEY COMMENT '主键，优惠券唯一标识',
    name VARCHAR(64) NOT NULL COMMENT '优惠券名称',
    type ENUM('full_reduction','discount') NOT NULL COMMENT '优惠券类型：full_reduction-满减券，discount-折扣券',
    value DECIMAL(10,2) NOT NULL COMMENT '优惠值：满减券为减免金额，折扣券为折扣率',
    min_amount DECIMAL(10,2) DEFAULT 0.00 COMMENT '最低消费金额（元）',
    start_time TIMESTAMP NOT NULL COMMENT '开始时间',
    end_time TIMESTAMP NOT NULL COMMENT '结束时间',
    total_count INT DEFAULT 0 COMMENT '发放总量',
    received_count INT DEFAULT 0 COMMENT '已领取数量',
    status TINYINT DEFAULT 1 COMMENT '状态：0-禁用，1-启用'
) COMMENT='优惠券表';
```

## 8. 用户体系说明

### 8.1 双用户体系设计

本系统采用**双表方案**来区分小程序用户和后台管理员，这是行业最佳实践：

| 用户类型 | 表名 | 登录方式 | 核心字段 | 安全要求 |
|---------|------|---------|---------|---------|
| 小程序用户 | t_user | 微信OAuth | openid, nickname, phone | 中 |
| 后台管理员 | t_admin | 账号密码 | username, password(BCrypt) | 高 |

### 8.2 角色权限体系

| 角色编码 | 角色名称 | 权限说明 |
|---------|---------|---------|
| super | 超级管理员 | 拥有系统最高权限，可操作所有功能 |
| admin | 管理员 | 拥有系统管理权限，可进行日常管理操作 |
| user | 普通用户 | 拥有基础操作权限，仅能查看部分数据 |

### 8.3 默认数据

系统初始化时会自动创建以下默认数据：

**默认角色：**
- 超级管理员 (super)
- 管理员 (admin)
- 普通用户 (user)

**默认管理员：**
- 用户名：`admin`
- 密码：`123456`（建议首次登录后修改）
