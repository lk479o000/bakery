# 软删除与文件上传设计文档

---

## 第一部分：软删除（Soft Delete）

### 1. 背景

当前项目所有数据库操作均为**物理删除**（直接从表中 DELETE 记录）。对于订单、用户、余额等核心业务数据，物理删除存在以下风险：

- 订单涉及支付、对账、售后纠纷，记录不可删除
- 用户注销后关联数据引用断链
- 优惠券活动结束后的核销/发放记录不可追溯
- 财务数据（余额变动）必须永久保留

### 2. 涉及的数据库表

| 表名 | 是否需要软删除 | 理由 |
|------|:------------:|------|
| `t_order` | ✅ 是 | 订单涉及支付对账，永久保留 |
| `t_order_item` | ❌ 否 | 跟随 `t_order`，主表软删除即关联不可见 |
| `t_user` | ✅ 是 | 保留用户数据引用完整性 |
| `t_product` | ✅ 是 | 历史订单快照需关联到商品 |
| `t_category` | ✅ 是 | 历史商品需关联到分类 |
| `t_coupon` | ✅ 是 | 保留优惠券发放和核销记录 |
| `t_user_coupon` | ❌ 否 | 跟随 `t_coupon` 主表 |
| `t_balance_record` | ✅ 是 | 财务数据，永久保留 |
| `t_store` | ✅ 是 | 历史订单关联门店 |
| `t_banner` | ✅ 是 | 运营记录可追溯 |
| `t_recharge_package` | ✅ 是 | 保留充值活动历史记录 |
| `t_cart` | ❌ 否 | 购物车数据无保留必要 |
| `t_address` | ✅ 是 | 历史订单关联地址快照 |
| `t_admin` | ✅ 是 | 管理员操作记录可追溯 |
| `t_role` / `t_admin_role` | ❌ 否 | 关联表跟随主表 |

### 3. 实现方案

#### 3.1 数据库层：增加 `deleted_at` 字段

为需要软删除的每张表增加 `deleted_at` 字段：

```sql
-- ========== t_order 订单表 ==========
ALTER TABLE t_order ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER pay_time;

-- ========== t_user 用户表 ==========
ALTER TABLE t_user ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER updated_at;

-- ========== t_product 商品表 ==========
ALTER TABLE t_product ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER created_at;

-- ========== t_category 分类表 ==========
ALTER TABLE t_category ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER can_express;

-- ========== t_coupon 优惠券表 ==========
ALTER TABLE t_coupon ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER status;

-- ========== t_balance_record 余额记录表 ==========
ALTER TABLE t_balance_record ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER create_time;

-- ========== t_store 门店表 ==========
ALTER TABLE t_store ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER created_at;

-- ========== t_banner 轮播图表 ==========
ALTER TABLE t_banner ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER status;

-- ========== t_recharge_package 充值套餐表 ==========
ALTER TABLE t_recharge_package ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER created_at;

-- ========== t_address 收货地址表 ==========
ALTER TABLE t_address ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER created_at;

-- ========== t_admin 管理员表 ==========
ALTER TABLE t_admin ADD COLUMN deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）' AFTER updated_at;
```

#### 3.2 ORM 层：启用 Sequelize `paranoid` 模式

在需要软删除的 Sequelize 模型中，新增 `deleted_at` 字段定义，并将 `timestamps` 设置为 `true`，同时开启 `paranoid: true`。

**现有模型模式**（以 `Product` 为例，当前 `timestamps: false`）：

```typescript
// server/models/product.model.ts
Product.init(
  {
    // ... 现有字段保持不变 ...
    deleted_at: {
      type: DataTypes.DATE,
      allowNull: true,
      comment: '删除时间（软删除）'
    }
  },
  {
    sequelize,
    tableName: 't_product',
    timestamps: true,          // 改为 true（sequelize 会自动管理）
    paranoid: true,            // 开启 paranoid 模式
    createdAt: 'created_at',   // 映射已有字段
    updatedAt: 'updated_at',   // 映射已有字段（如果表没有此字段则不设置）
    deletedAt: 'deleted_at'    // 软删除时间戳
  }
);
```

**⚠️ 注意事项**：

- 对于**没有 `updated_at` 字段**的表（如 `t_coupon`、`t_store`、`t_banner`、`t_recharge_package`、`t_address`、`t_balance_record`），不设置 `updatedAt`，Sequelize 不会自动写入该字段。
- 对于**有 `updated_at` 字段**的表（如 `t_user`、`t_admin`），设置 `updatedAt: 'updated_at'` 以复用。
- 部分表的时间字段命名不同（如 `t_order` 使用 `create_time` 而非 `created_at`），需在模型中正确映射。

#### 3.3 各模型 paranoid 配置对照表

| 模型文件 | 表名 | createdAt 映射 | updatedAt 映射 | deletedAt 映射 |
|---------|------|:---:|:---:|:---:|
| `order.model.ts` | `t_order` | `create_time` | 不设置（无此字段） | `deleted_at` |
| `user.model.ts` | `t_user` | `created_at` | `updated_at` | `deleted_at` |
| `product.model.ts` | `t_product` | `created_at` | 不设置（无此字段） | `deleted_at` |
| `category.model.ts` | `t_category` | 不设置（无此字段） | 不设置（无此字段） | `deleted_at` |
| `coupon.model.ts` | `t_coupon` | 不设置（无此字段） | 不设置（无此字段） | `deleted_at` |
| `balance.model.ts` | `t_balance_record` | `create_time` | 不设置（无此字段） | `deleted_at` |
| `store.model.ts` | `t_store` | `created_at` | 不设置（无此字段） | `deleted_at` |
| `banner.model.ts` | `t_banner` | 不设置（无此字段） | 不设置（无此字段） | `deleted_at` |
| `recharge-package.model.ts` | `t_recharge_package` | `created_at` | 不设置（无此字段） | `deleted_at` |
| `address.model.ts` | `t_address` | `created_at` | 不设置（无此字段） | `deleted_at` |
| `admin.model.ts` | `t_admin` | `created_at` | `updated_at` | `deleted_at` |

#### 3.4 行为变更说明

启用 `paranoid: true` 后，Sequelize 自动改变以下行为：

| 操作 | 原始行为 | paranoid 行为 |
|------|---------|--------------|
| `Model.destroy()` | 物理 DELETE | 设置 `deleted_at = NOW()` |
| `Model.findAll()` | 返回所有记录 | 自动加 `WHERE deleted_at IS NULL` |
| `Model.findOne()` | 返回匹配记录 | 自动加 `WHERE deleted_at IS NULL` |
| `Model.restore()` | 不支持 | 设置 `deleted_at = NULL`，恢复记录 |

> 💡 **强制物理删除**：如需真正删除记录，使用 `Model.destroy({ force: true })`。

### 4. 更新后的完整 schema.sql 修改

以 `t_order` 为例，在 schema.sql 中增加 `deleted_at` 字段后的建表语句：

```sql
CREATE TABLE t_order (
    id VARCHAR(32) PRIMARY KEY COMMENT '主键，订单唯一标识',
    user_id VARCHAR(32) NOT NULL COMMENT '用户ID，关联t_user.id',
    order_no VARCHAR(32) UNIQUE NOT NULL COMMENT '订单编号',
    type ENUM('store','recharge','payment','coupon') DEFAULT 'store' COMMENT '订单类型',
    status TINYINT DEFAULT 0 COMMENT '订单状态：0-待支付，1-已支付，2-制作中，3-待取餐，4-已完成，5-已取消',
    total_amount DECIMAL(10,2) NOT NULL COMMENT '订单总金额（元）',
    discount_amount DECIMAL(10,2) DEFAULT 0.00 COMMENT '优惠金额（元）',
    pay_amount DECIMAL(10,2) NOT NULL COMMENT '实付金额（元）',
    pay_type ENUM('wechat','balance') COMMENT '支付方式',
    remark VARCHAR(255) COMMENT '订单备注',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    pay_time TIMESTAMP NULL COMMENT '支付时间',
    deleted_at TIMESTAMP NULL COMMENT '删除时间（软删除）',
    INDEX idx_user_id (user_id),
    INDEX idx_order_no (order_no)
) COMMENT='订单表';
```

## 第二部分：文件上传功能

### 1. 背景

当前项目中，所有图片（商品图、轮播图、分类图标、用户头像等）均以**外部 URL 字符串**形式存储在数据库字段中。种子数据使用 `https://example.com/images/bread1.jpg` 占位符，无法满足真实运营需求。

管理后台的商品编辑、轮播图管理、门店管理等模块，以及小程序端用户头像更换，都需要文件上传能力。

### 2. 设计方案：本地 + OSS 双模式

项目需要同时支持两种存储模式，通过环境变量切换：

```
UPLOAD_MODE=local   # 上传到服务器本地 public/images/
UPLOAD_MODE=oss     # 上传到对象存储 OSS
```

| 模式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **local** | 开发环境、单机部署 | 零成本、配置简单 | 多实例不共享，占用磁盘 |
| **oss** | 生产环境、集群部署 | CDN加速、弹性扩容、高可用 | 需要费用、需配置 |

### 3. OSS 厂商支持

同时支持主流对象存储服务，通过 `UPLOAD_OSS_PROVIDER` 环境变量切换：

| 值 | OSS 厂商 |
|:--:|---------|
| `aliyun` | 阿里云 OSS |
| `tencent` | 腾讯云 COS |
| `qiniu` | 七牛云 Kodo |
| `aws` | AWS S3（兼容 MinIO） |

### 4. 配置

#### 5.1 图片上传接口

| 项目 | 内容 |
|------|------|

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |

**成功响应**：

```json
  "message": "上传成功",

    "filename": "a1b2c3d4.jpg",
    "size": 245760,
    "mimeType": "image/jpeg"
  }

```

**错误响应**：

```json
{
  "code": 400,
  "message": "仅支持 jpeg、png、gif、webp、svg 格式的图片",
  "data": null
}
```

#### 5.2 文件组织规则

```
{基础路径} / {业务目录} / {年} / {月} / {日} / {uuid}.{扩展名}

示例：
- 商品图片： /images/product/2025/05/15/a1b2c3d4.jpg
- 轮播图：   /images/banner/2025/05/15/e5f6g7h8.jpg
- 分类图标： /images/category/2025/05/15/i9j0k1l2.png
- 用户头像： /images/avatar/2025/05/15/m3n4o5p6.jpg
```

> 在请求参数中增加 `dir` 字段（可选值：`product`、`banner`、`category`、`avatar`、`store`），默认为 `product`。

### 7. 核心代码示例

#### 7.1 配置（`config/upload.config.ts`）

```typescript
interface UploadConfig {
  mode: 'local' | 'oss';
  localDir: string;
  maxSize: number; // bytes
  allowTypes: string[];
  oss: {
    provider: 'aliyun' | 'tencent' | 'qiniu' | 'aws';
    // 各家 OSS 配置字段...
  } | null;
}

const uploadConfig: UploadConfig = {
  mode: (process.env.UPLOAD_MODE as 'local' | 'oss') || 'local',
  localDir: process.env.UPLOAD_LOCAL_DIR || 'public/images',
  maxSize: (parseInt(process.env.UPLOAD_MAX_SIZE || '10', 10)) * 1024 * 1024,
  allowTypes: (process.env.UPLOAD_ALLOW_TYPES || 'image/jpeg,image/png,image/gif,image/webp,image/svg+xml').split(','),
  oss: process.env.UPLOAD_MODE === 'oss' ? {
    provider: (process.env.UPLOAD_OSS_PROVIDER as 'aliyun' | 'tencent' | 'qiniu' | 'aws') || 'aliyun',
  } : null
};

export default uploadConfig;
```

#### 7.2 multer 中间件（`middleware/upload.middleware.ts`）

```typescript
import multer from 'multer';
import path from 'path';
import { v4 as uuidv4 } from 'uuid';
import uploadConfig from '../config/upload.config';

const storage = multer.diskStorage({
  destination: (_req, _file, cb) => {
    const date = new Date();
    const dir = path.join(
      uploadConfig.localDir,
      date.getFullYear().toString(),
      (date.getMonth() + 1).toString().padStart(2, '0'),
      date.getDate().toString().padStart(2, '0')
    );
    // 确保目录存在
    const fs = require('fs');
    fs.mkdirSync(dir, { recursive: true });
    cb(null, dir);
  },
  filename: (_req, file, cb) => {
    const ext = path.extname(file.originalname);
    cb(null, uuidv4() + ext);
  }
});

const fileFilter = (_req: any, file: Express.Multer.File, cb: multer.FileFilterCallback) => {
  if (uploadConfig.allowTypes.includes(file.mimetype)) {
    cb(null, true);
  } else {
    cb(new Error('不支持的文件类型'));
  }
};

export const uploadMiddleware = multer({
  storage,
  fileFilter,
  limits: { fileSize: uploadConfig.maxSize }
});
```

#### 7.4 控制器（`controllers/upload.controller.ts`）

```typescript
import { Request, Response, NextFunction } from 'express';
import uploadService from '../services/upload/upload.service';

export const uploadImage = async (req: Request, res: Response, next: NextFunction) => {
  try {
    if (!req.file) {
      return res.status(400).json({ code: 400, message: '请选择要上传的文件', data: null });
    }

    const result = await uploadService.upload(req.file);
    
    res.json({
      code: 0,
      message: '上传成功',
      data: result
    });
  } catch (error: any) {
    if (error.message === '不支持的文件类型') {
      return res.status(400).json({ code: 400, message: '仅支持 jpeg、png、gif、webp、svg 格式的图片', data: null });
    }
    next(error);
  }
};
```

#### 7.5 上传服务入口（`services/upload/upload.service.ts`）

```typescript
import uploadConfig from '../../config/upload.config';
import { localUploader } from './local.uploader';
import { aliyunUploader } from './aliyun.uploader';
import { tencentUploader } from './tencent.uploader';
import { qiniuUploader } from './qiniu.uploader';
import { awsUploader } from './aws.uploader';

interface UploadResult {
  url: string;
  filename: string;
  size: number;
  mimeType: string;
}

const upload = async (file: Express.Multer.File): Promise<UploadResult> => {
  const { mode, oss } = uploadConfig;

  if (mode === 'local') {
    return localUploader.upload(file);
  }

  // OSS 模式
  if (!oss) {
    throw new Error('OSS 配置未设置');
  }

  switch (oss.provider) {
    case 'aliyun':  return aliyunUploader.upload(file);
    case 'tencent': return tencentUploader.upload(file);
    case 'qiniu':   return qiniuUploader.upload(file);
    case 'aws':     return awsUploader.upload(file);
    default:        throw new Error(`不支持的 OSS 提供商: ${oss.provider}`);
  }
};

export default { upload };
```

#### 7.6 本地上传器（`services/upload/local.uploader.ts`）

```typescript
import path from 'path';

const upload = async (file: Express.Multer.File) => {
  // multer 已将文件写入 localDir，file.path 即完整磁盘路径
  // 转换为可访问 URL（去掉 server/ 前缀的逻辑由具体部署决定）
  const relativePath = file.path.replace(/\\/g, '/');
  const match = relativePath.match(/public\/(images\/.+)$/);

  const url = match 
    ? `/images/${match[1].replace('images/', '').replace(/^\//, '')}`
    : relativePath;

  // 生产环境应拼接完整域名
  const baseUrl = process.env.BASE_URL || `http://localhost:${process.env.PORT || 8150}`;
  
  return {
    url: `${baseUrl}${url}`,
    filename: file.filename,
    size: file.size,
    mimeType: file.mimetype
  };
};

export const localUploader = { upload };
```

#### 7.7 阿里云 OSS 上传器（`services/upload/aliyun.uploader.ts`）

```typescript
import OSS from 'ali-oss';
import path from 'path';
import fs from 'fs';

let client: OSS | null = null;

const getClient = (): OSS => {
  if (client) return client;
  
  client = new OSS({
    region: process.env.OSS_ALIYUN_REGION || 'oss-cn-hangzhou',
    accessKeyId: process.env.OSS_ALIYUN_ACCESS_KEY_ID!,
    accessKeySecret: process.env.OSS_ALIYUN_ACCESS_KEY_SECRET!,
    bucket: process.env.OSS_ALIYUN_BUCKET!,
    endpoint: process.env.OSS_ALIYUN_ENDPOINT
  });
  
  return client;
};

const upload = async (file: Express.Multer.File) => {
  const oss = getClient();
  
  // 构建 OSS 存储路径
  const date = new Date();
  const datePath = `${date.getFullYear()}/${(date.getMonth() + 1).toString().padStart(2, '0')}/${date.getDate().toString().padStart(2, '0')}`;
  const key = `images/${datePath}/${file.filename}`;
  
  const result = await oss.put(key, file.path);

  // 本地上传 OSS 成功后删除临时文件
  fs.unlink(file.path, () => {});

  const cdnDomain = process.env.OSS_ALIYUN_CDN_DOMAIN;
  const url = cdnDomain
    ? `${cdnDomain}/${key}`
    : result.url;

  return {
    url,
    filename: file.filename,
    size: file.size,
    mimeType: file.mimetype
  };
};

export const aliyunUploader = { upload };
```

### 10. 安全策略

| 策略 | 说明 |
|------|------|
| **文件类型白名单** | 仅允许 `image/jpeg, image/png, image/gif, image/webp` 上传 |
| **文件大小限制** | 默认 10MB，防止大文件攻击 |
| **魔数校验** | 校验文件头字节，防止伪造 MIME Type |
| **文件名防冲突** | 使用 UUID 重命名，防止路径穿越 |
| **认证保护** | 管理后台接口需要 Admin Bearer Token |
| **OSS 私有读写** | OSS Bucket 设为私有，通过签名 URL 或 CDN 回源访问 |
| **本地模式防护** | `public/images/` 目录设只读权限，禁止执行脚本 |
