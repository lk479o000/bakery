# pages/order-detail/order-detail 交互流程说明

## 1. 页面初始化流程

1. **进入页面**：用户从订单列表页面点击订单进入订单详情页面，或从下单页面跳转而来
2. **参数获取**：从URL参数中获取订单ID（`orderId`）或操作步骤（如 `step=confirm`）
3. **登录检查**：检查用户登录状态，未登录则跳转到登录页面
4. **初始化数据**：设置初始加载状态，准备请求订单详情数据

## 2. 加载订单详情流程

1. **发起请求**：调用 `getOrderDetail` API，传入订单ID获取订单详情
2. **加载状态**：显示加载动画，提示用户正在加载
3. **数据处理**：
   - 成功：解析返回的订单详情数据，包括订单基本信息、商品列表、价格信息等
   - 失败：显示错误提示，允许用户重试
4. **渲染页面**：根据订单详情数据渲染页面，包括：
   - 订单状态、订单号、创建时间等基本信息
   - 商品列表（商品名称、图片、价格、数量）
   - 价格明细（总金额、优惠金额、实付金额）
   - 支付方式、备注等信息

## 3. 支付流程

1. **条件判断**：当订单状态为待支付时，显示支付按钮
2. **支付方式选择**：用户选择支付方式（微信支付或余额支付）
3. **发起支付**：点击支付按钮，调用 `payOrder` API
4. **支付处理**：
   - 微信支付：调用微信支付SDK，调起支付界面
   - 余额支付：直接扣除用户余额
5. **支付结果**：
   - 成功：显示支付成功提示，更新订单状态为已支付
   - 失败：显示支付失败提示，允许用户重新支付

## 4. 取消订单流程

1. **条件判断**：当订单状态为待支付时，显示取消订单按钮
2. **确认操作**：点击取消订单按钮后，弹出确认对话框，提示用户确认取消
3. **发起取消**：用户确认后，调用 `cancelOrder` API
4. **取消结果**：
   - 成功：显示取消成功提示，更新订单状态为已取消
   - 失败：显示取消失败提示，允许用户重试

## 5. 再来一单流程

1. **显示按钮**：在订单详情页面底部显示"再来一单"按钮
2. **点击操作**：用户点击"再来一单"按钮
3. **商品处理**：将当前订单中的商品信息添加到购物车
4. **页面跳转**：跳转到下单页面（`/pages/order/order`），用户可以在该页面修改商品数量或添加其他商品

## 6. 退款流程

1. **条件判断**：当订单状态为已支付但未完成时，显示退款按钮
2. **确认操作**：点击退款按钮后，弹出确认对话框，提示用户确认退款
3. **发起退款**：用户确认后，调用退款相关API
4. **退款结果**：
   - 成功：显示退款成功提示，更新订单状态为已退款
   - 失败：显示退款失败提示，允许用户重试

## 7. UI状态切换

1. **加载状态**：
   - 页面初始化时显示加载动画
   - API请求过程中显示加载提示
   - 加载完成后隐藏加载动画
2. **订单状态对应UI**：
   - 待支付：显示支付按钮和取消订单按钮
   - 已支付：显示订单详情，可能显示退款按钮
   - 已完成：显示订单详情，无操作按钮
   - 已取消：显示订单详情，无操作按钮
   - 已退款：显示订单详情，无操作按钮
3. **操作反馈**：
   - 成功操作：显示成功提示（如"支付成功"、"取消成功"）
   - 失败操作：显示失败提示（如"支付失败"、"取消失败"）
   - 确认操作：弹出确认对话框，避免用户误操作
4. **页面刷新**：
   - 操作成功后自动刷新订单详情
   - 提供手动刷新功能，允许用户刷新订单状态

## 8. 异常处理

1. **网络错误**：显示网络错误提示，允许用户重试
2. **登录过期**：跳转到登录页面，登录后返回订单详情
3. **权限不足**：显示权限不足提示，引导用户返回
4. **数据异常**：显示数据异常提示，允许用户返回订单列表

## 9. 页面退出

1. **返回按钮**：点击返回按钮或使用系统返回，返回上一页
2. **导航栏**：提供返回订单列表的入口
3. **分享功能**：支持分享订单详情给好友
