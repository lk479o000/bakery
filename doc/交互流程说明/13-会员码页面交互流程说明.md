# 会员码页面（pages/member-code/member-code）交互流程说明

## 一、页面初始化流程

```
┌─────────────────────────────────────────────────────────────┐
│                     页面初始化流程                           │
└─────────────────────────────────────────────────────────────┘

onLoad
  │
  ├─> 1. 初始化页面数据
  │     - memberCode: null (会员码数据)
  │     - loading: true (加载状态)
  │     - error: null (错误信息)
  │     - countdown: 0 (倒计时秒数)
  │     - lastRefreshTime: 0 (上次刷新时间)
  │
  ├─> 2. 检查登录状态
  │     - 调用 checkLoginStatus()
  │     - 未登录：显示未登录状态
  │     - 已登录：调用 loadMemberCode() 加载会员码
  │
  └─> 3. 页面渲染完成
```

### 数据结构设计

```typescript
interface PageData {
  // 会员码相关
  memberCode: {
    code: string;        // 会员码
    expireTime: number;  // 过期时间戳
    refreshTime: number; // 刷新时间戳
  } | null;
  
  // 状态相关
  loading: boolean;     // 加载状态
  error: string | null; // 错误信息
  
  // 倒计时相关
  countdown: number;    // 倒计时秒数
  lastRefreshTime: number; // 上次刷新时间
  
  // 登录状态
  isLoggedIn: boolean;  // 是否已登录
}
```

## 二、加载会员码数据流程

```
┌─────────────────────────────────────────────────────────────┐
│                   加载会员码数据流程                           │
└─────────────────────────────────────────────────────────────┘

loadMemberCode()
  │
  ├─> 1. 显示加载状态
  │     this.setData({ loading: true, error: null })
  │
  ├─> 2. 发起请求获取会员码
  │     API: GET /api/member/code
  │     Headers: { Authorization: 'Bearer token' }
  │
  ├─> 3. 处理响应
  │     Success:
  │       - 保存会员码数据
  │       - 计算倒计时
  │       - 启动倒计时
  │       - 记录刷新时间
  │     Fail:
  │       - 显示错误提示
  │       - 处理错误类型
  │
  ├─> 4. 更新页面状态
  │     this.setData({
  │       memberCode: response.data,
  │       loading: false,
  │       countdown: calculateCountdown(response.data.expireTime),
  │       lastRefreshTime: Date.now()
  │     })
  │
  └─> 5. 启动倒计时
        this.startCountdown()
```

## 三、会员码刷新流程

```
┌─────────────────────────────────────────────────────────────┐
│                     会员码刷新流程                           │
└─────────────────────────────────────────────────────────────┘

用户点击刷新按钮
  │
  ├─> 1. 检查刷新频率
  │     - 上次刷新时间与当前时间间隔小于10秒：提示"刷新过于频繁"
  │     - 时间间隔大于等于10秒：继续流程
  │
  ├─> 2. 调用 loadMemberCode() 重新加载会员码
  │
  ├─> 3. 显示刷新成功提示
  │     wx.showToast({ title: '会员码已刷新' })
  │
  └─> 4. 页面显示新的会员码
```

## 四、会员码失效与重新获取流程

```
┌─────────────────────────────────────────────────────────────┐
│                 会员码失效与重新获取流程                      │
└─────────────────────────────────────────────────────────────┘

倒计时结束或会员码已过期
  │
  ├─> 1. 显示过期状态
  │     - 会员码变为灰色
  │     - 显示"已过期"提示
  │     - 倒计时显示"00:00"
  │
  ├─> 2. 自动重新获取
  │     - 调用 loadMemberCode()
  │     - 显示加载状态
  │
  ├─> 3. 处理获取结果
  │     Success:
  │       - 显示新会员码
  │       - 重新启动倒计时
  │     Fail:
  │       - 显示错误提示
  │       - 提供手动刷新按钮
  │
  └─> 4. 页面更新
```

## 五、未登录状态处理流程

```

│                   未登录状态处理流程                           │
└─────────────────────────────────────────────────────────────┘

  │
  ├─> 3. 登录成功返回

  │

```

## 六、会员码加载失败与错误提示流程

```
┌─────────────────────────────────────────────────────────────┐
│                会员码加载失败与错误提示流程                     │
└─────────────────────────────────────────────────────────────┘

loadMemberCode() 失败
  │
  ├─> 1. 分析错误类型
  │     - 401 Unauthorized: 未登录或登录过期
  │     - 403 Forbidden: 无权限获取会员码
  │     - 500 Internal Server Error: 服务端错误
  │     - 网络错误: 网络连接问题
  │
  ├─> 2. 显示对应错误提示
  │     - 401: 跳转登录页
  │     - 403: 提示"您暂无会员码权限"
  │     - 500: 提示"系统错误，请稍后重试"
  │     - 网络错误: 提示"网络连接失败，请检查网络"
  │
  ├─> 3. 更新页面状态
  │     this.setData({
  │       loading: false,
  │       error: errorMessage,
  │       memberCode: null
  │     })
  │
  ├─> 4. 提供重试按钮
  │     - 点击后重新调用 loadMemberCode()
  │
  └─> 5. 记录错误日志
        console.error('会员码加载失败:', error)
```

## 八、关键接口汇总

| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 获取会员码 | /api/member/code | GET | 需要登录，返回会员码及过期时间 |
| 刷新会员码 | /api/member/code | GET | 与获取会员码使用同一接口 |

## 九、异常处理策略

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示"网络连接失败，请检查网络"，提供重试按钮 |
| 未登录 | 显示登录按钮，点击跳转登录页 |
| 无权限 | 显示"您暂无会员码权限"提示 |
| 服务端错误 | 显示"系统错误，请稍后重试"，提供重试按钮 |
| 刷新频繁 | 显示"刷新过于频繁，请稍后再试"提示 |
| 会员码过期 | 自动重新获取，若失败显示错误提示 |

## 十、性能优化策略

1. **缓存策略**：
   - 本地缓存会员码数据，减少重复请求
   - 缓存时间不超过会员码过期时间

2. **刷新频率控制**：
   - 限制手动刷新频率（至少10秒间隔）
   - 自动刷新仅在会员码过期时触发

3. **请求优化**：
   - 使用防抖处理，避免频繁请求
   - 网络错误后使用指数退避策略重试

4. **UI优化**：
   - 加载状态使用骨架屏
   - 会员码显示使用Canvas绘制，提高性能
   - 倒计时动画使用requestAnimationFrame

5. **体验优化**：
   - 会员码即将过期时（如最后30秒）显示提示
   - 网络恢复后自动重试加载
   - 登录成功后自动返回并加载会员码
