# 地址编辑页（pages/address-edit/address-edit）交互流程说明

## 一、页面初始化流程

```
┌─────────────────────────────────────────────────────────────┐
│                     页面初始化流程                           │
└─────────────────────────────────────────────────────────────┘

onLoad(options)
  │
  ├─> 1. 初始化页面数据
  │     - id: options.id (地址ID，编辑模式下存在)
  │     - isEditMode: !!options.id (是否为编辑模式)
  │     - formData: {
  │         name: '',
  │         phone: '',
  │         province: '',
  │         city: '',
  │         district: '',
  │         detail: '',
  │         isDefault: false
  │       }
  │     - formErrors: {}
  │     - loading: false
  │
  ├─> 2. 判断页面模式
  │     ├─> 2.1 编辑模式 (options.id 存在)
  │     │     - 调用 loadAddressDetail() 加载地址详情
  │     │
  │     └─> 2.2 新增模式 (options.id 不存在)
  │           - 保持默认表单数据
  │
  └─> 3. 页面渲染完成
```

### 数据结构设计

```typescript
interface PageData {
  // 页面状态
  isEditMode: boolean; // 是否为编辑模式
  addressId: string | null; // 地址ID
  loading: boolean; // 加载状态
  
  // 表单数据
  formData: {
    name: string; // 收货人姓名
    phone: string; // 手机号码
    province: string; // 省份
    city: string; // 城市
    district: string; // 区县
    detail: string; // 详细地址
    isDefault: boolean; // 是否默认地址
  };
  
  // 表单错误
  formErrors: {
    name?: string;
    phone?: string;
    province?: string;
    city?: string;
    district?: string;
    detail?: string;
  };
}
```

## 二、加载已有地址详情流程（编辑模式）

```
┌─────────────────────────────────────────────────────────────┐
│                  加载已有地址详情流程                         │
└─────────────────────────────────────────────────────────────┘

loadAddressDetail(addressId)
  │
  ├─> 1. 显示加载状态
  │     this.setData({ loading: true })
  │
  ├─> 2. 发起请求获取地址详情
  │     API: GET /api/address/detail
  │     Params: { addressId }
  │
  ├─> 3. 处理响应
  │     Success:
  │       - 更新表单数据
  │       - 隐藏加载状态
  │     Fail:
  │       - 显示错误提示
  │       - 隐藏加载状态
  │       - 跳转到地址列表页
  │

        this.setData({

          loading: false

异常处理：

  - 地址不存在：显示"地址不存在"
  - 权限错误：显示"无权限访问此地址"
```

## 三、表单输入与校验流程

```
┌─────────────────────────────────────────────────────────────┐
│                    表单输入与校验流程                         │
└─────────────────────────────────────────────────────────────┘

用户输入表单字段
  │
  ├─> 1. 实时更新表单数据
  │     this.setData({ 
  │       ['formData.' + field]: value 
  │     })
  │
  ├─> 2. 实时校验（可选）
  │     - 姓名：非空校验
  │     - 手机号：格式校验
  │     - 地址：非空校验
  │
  └─> 3. 更新错误提示（如有）
        this.setData({ 
          ['formErrors.' + field]: errorMessage 
        })

用户点击保存按钮
  │
  ├─> 1. 执行完整表单校验
  │     const errors = validateForm(this.data.formData);
  │
  ├─> 2. 检查校验结果
  │     ├─> 2.1 校验通过
  │     │     - 继续保存流程
  │     │
  │     └─> 2.2 校验失败
  │           - 显示错误提示
  │           - 滚动到第一个错误字段
  │
  └─> 3. 阻止继续执行

表单校验规则：
  - 姓名：必填，长度1-20
  - 手机号：必填，11位数字，格式正确
  - 省份：必填
  - 城市：必填
  - 区县：必填
  - 详细地址：必填，长度1-200
  - 是否默认：可选
```

### 表单校验函数

```typescript
function validateForm(formData: any): Record<string, string> {
  const errors: Record<string, string> = {};
  
  // 姓名校验
  if (!formData.name.trim()) {
    errors.name = '请输入收货人姓名';
  } else if (formData.name.length > 20) {
    errors.name = '姓名长度不能超过20个字符';
  }
  
  // 手机号校验
  const phoneRegex = /^1[3-9]\d{9}$/;
  if (!formData.phone.trim()) {
    errors.phone = '请输入手机号码';
  } else if (!phoneRegex.test(formData.phone)) {
    errors.phone = '请输入正确的手机号码';
  }
  
  // 地址校验
  if (!formData.province) {
    errors.province = '请选择省份';
  }
  if (!formData.city) {
    errors.city = '请选择城市';
  }
  if (!formData.district) {
    errors.district = '请选择区县';
  }
  if (!formData.detail.trim()) {
    errors.detail = '请输入详细地址';
  } else if (formData.detail.length > 200) {
    errors.detail = '详细地址长度不能超过200个字符';
  }
  
  return errors;
}
```

```
└─────────────────────────────────────────────────────────────┘

  │     │     API: PUT /api/address/update
  │     │     Params: {
  │     │       addressId: string,
  │     │       name: string,
  │     │       phone: string,
  │     │       province: string,
  │     │       city: string,
  │     │       district: string,
  │     │       detail: string,
  │     │       isDefault: boolean
  │     │     }
  │     │
  │     └─> 3.2 新增模式
  │           API: POST /api/address/add
  │           Params: {
  │             name: string,
  │             phone: string,
  │             province: string,
  │             city: string,
  │             district: string,
  │             detail: string,
  │             isDefault: boolean
```

## 八、关键接口汇总

| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 获取地址详情 | /api/address/detail | GET | 需要登录，根据地址ID获取详情 |
| 添加地址 | /api/address/add | POST | 需要登录，新增收货地址 |
| 更新地址 | /api/address/update | PUT | 需要登录，更新收货地址 |
| 删除地址 | /api/address/delete | DELETE | 需要登录，删除收货地址 |
| 设置默认地址 | /api/address/default | PUT | 需要登录，设置默认地址 |

## 九、异常处理策略

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示"网络错误，请重试"，保留表单数据 |
| 未登录 | 跳转登录页，登录成功后返回 |
| 表单验证错误 | 显示具体字段错误提示，滚动到错误字段 |
| 地址不存在 | 显示"地址不存在"，跳转到地址列表页 |
| 权限错误 | 显示"无权限操作此地址"，跳转到地址列表页 |
| 服务端错误 | 显示服务端返回的错误信息或"保存失败，请重试" |
