# 联系客服页面（pages/service/service）交互流程说明

## 一、页面初始化流程

```
┌─────────────────────────────────────────────────────────────┐
│                     页面初始化流程                           │
└─────────────────────────────────────────────────────────────┘

onLoad
  │
  ├─> 1. 初始化页面数据
  │     - serviceInfo: {} (客服信息)
  │     - loading: true (加载状态)
  │     - error: false (错误状态)
  │
  ├─> 2. 加载客服信息
  │     API: GET /api/service/info
  │     - 成功后更新 serviceInfo
  │     - 失败后显示错误状态
  │
  ├─> 3. 检查微信客服功能可用性
  │     - 调用 wx.getSystemInfoSync() 检查基础库版本
  │     - 验证是否支持 wx.openCustomerServiceChat
  │
  └─> 4. 页面渲染完成
        this.setData({ loading: false })
```

### 数据结构设计

```typescript
interface PageData {
  // 客服信息
  serviceInfo: {
    phone: string;        // 客服电话
    wechat: string;       // 微信号
    onlineService: boolean; // 是否支持在线客服
    serviceTime: string;  // 服务时间
  };
  
  // 页面状态
  loading: boolean;       // 加载状态
  error: boolean;         // 错误状态
  errorMessage: string;   // 错误信息
  
  // 操作状态
  copySuccess: boolean;   // 复制成功状态
}
```

## 二、展示客服联系方式流程

```
┌─────────────────────────────────────────────────────────────┐
│                  展示客服联系方式流程                         │
└─────────────────────────────────────────────────────────────┘

页面渲染时
  │
  ├─> 1. 检查 serviceInfo 是否存在
  │     - 存在：显示联系方式列表
  │     - 不存在：显示加载中或错误状态
  │
  ├─> 2. 渲染联系方式列表
  │     ├─> 电话客服
  │     │     - 显示电话号码
  │     │     - 显示拨打电话按钮
  │     │     - 显示服务时间
  │     │
  │     ├─> 微信客服
  │     │     - 显示微信号
  │     │     - 显示复制按钮
  │     │     - 显示二维码（可选）
  │     │
  │     └─> 在线客服
  │           - 显示在线客服按钮
  │           - 根据支持状态显示或隐藏
  │
  └─> 3. 处理不同状态
        - 加载中：显示加载动画
        - 错误：显示错误提示和重试按钮
        - 空数据：显示空状态提示
```

## 三、拨打电话流程

```
┌─────────────────────────────────────────────────────────────┐
│                     拨打电话流程                             │
└─────────────────────────────────────────────────────────────┘

用户点击拨打电话按钮
  │
  ├─> 1. 检查电话号码是否有效
  │     - 无效：显示错误提示
  │     - 有效：继续流程
  │
  ├─> 2. 调用微信拨打电话 API
  │     wx.makePhoneCall({
  │       phoneNumber: serviceInfo.phone,
  │       success: function() {
  │         console.log('拨打电话成功');
  │       },
  │       fail: function() {
  │         console.log('拨打电话失败');
  │       }
  │     })
  │
  ├─> 3. 处理调用结果
  │     - 成功：系统会自动打开拨号界面
  │     - 失败：显示错误提示
  │
  └─> 4. 记录操作日志（可选）
        API: POST /api/service/log
        Params: { action: 'call', phone: serviceInfo.phone }
```

## 六、操作失败与错误提示流程

```
┌─────────────────────────────────────────────────────────────┐
│                 操作失败与错误提示流程                         │
└─────────────────────────────────────────────────────────────┘

操作失败时
  │
  ├─> 1. 网络错误
  │     - 显示网络错误提示
  │     - 提供重试按钮
  │     - 点击重试：重新加载客服信息
  │
  ├─> 2. API 调用失败
  │     - 显示具体错误信息
  │     - 提供重试按钮
  │     - 点击重试：重新执行操作
  │
  ├─> 3. 权限不足
  │     - 显示权限不足提示
  │     - 引导用户授权
  │
  ├─> 4. 功能不支持
  │     - 显示功能不支持提示
  │     - 提供替代方案
  │
  └─> 5. 其他错误
        - 显示通用错误提示
        - 提供重试或返回按钮
```

### 错误提示类型

| 错误类型 | 提示内容 | 处理方式 |
|---------|---------|----------|
| 网络错误 | 网络连接失败，请检查网络设置 | 显示重试按钮 |
| 加载失败 | 客服信息加载失败 | 显示重试按钮 |
| 拨打电话失败 | 拨打电话失败，请手动拨打 | 显示电话号码 |
| 复制失败 | 复制失败，请手动复制 | 显示联系方式 |
| 客服会话失败 | 打开客服会话失败 | 显示其他联系方式 |
| 功能不支持 | 当前版本不支持此功能 | 显示其他联系方式 |

## 八、关键接口汇总

| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 获取客服信息 | /api/service/info | GET | 无需登录，获取客服联系方式 |
| 记录操作日志 | /api/service/log | POST | 记录用户操作行为 |

### 接口响应格式

#### 获取客服信息响应
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "phone": "10086",
    "wechat": "service_wechat",
    "onlineService": true,
    "serviceTime": "周一至周日 9:00-18:00"
  }
}
```

#### 记录操作日志响应
```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

## 九、异常处理策略

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示网络错误提示，提供重试按钮 |
| API 调用失败 | 根据错误码显示对应提示，提供重试选项 |
| 权限不足 | 引导用户授权相关权限 |
| 功能不支持 | 显示替代联系方式，引导用户使用其他方式 |
| 空数据 | 显示空状态提示，提供刷新按钮 |
| 用户取消操作 | 不做特殊处理，保持当前状态 |

## 十、页面优化建议

1. **缓存策略**：缓存客服信息到本地，减少重复请求
2. **预加载**：页面进入时预加载客服信息
3. **错误重试**：为所有网络操作添加重试机制
4. **用户体验**：
   - 添加加载动画
   - 优化错误提示文案
   - 提供清晰的操作反馈
5. **兼容性**：
   - 针对不同基础库版本做兼容处理
   - 提供降级方案
6. **性能优化**：
   - 减少不必要的渲染
   - 优化图片加载
   - 使用合适的动画效果

## 十一、开发注意事项

1. **微信 API 调用**：确保在合适的时机调用微信 API，遵守微信小程序的使用规范
2. **权限处理**：处理好拨打电话等需要用户授权的操作
3. **错误处理**：完善的错误处理机制，确保页面不会崩溃
4. **用户体验**：注重细节，提供流畅的交互体验
5. **代码规范**：遵循项目的代码规范，保持代码整洁
6. **测试**：在不同设备和网络环境下测试页面功能
7. **安全**：保护用户隐私，不泄露敏感信息
