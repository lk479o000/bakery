# 我的优惠券（pages/my-coupon/my-coupon）交互流程说明

## 一、页面初始化流程

```
┌─────────────────────────────────────────────────────────────┐
│                     页面初始化流程                           │
└─────────────────────────────────────────────────────────────┘

onLoad
  │
  ├─> 1. 初始化页面数据
  │     - currentStatus: 0 (当前选中状态：0-未使用, 1-已使用, 2-已过期)
  │     - couponList: [] (优惠券列表)
  │     - loading: false (加载状态)
  │     - empty: false (空状态)
  │
  ├─> 2. 检查登录状态
  │     - 已登录：继续流程
  │     - 未登录：显示未登录状态
  │
  ├─> 3. 加载优惠券列表
  │     - 调用 loadCouponList() 加载默认状态（未使用）的优惠券
  │
  └─> 4. 页面渲染完成
```

### 数据结构设计

```typescript
interface PageData {
  // 状态相关
  currentStatus: number; // 0-未使用, 1-已使用, 2-已过期
  loading: boolean; // 加载状态
  empty: boolean; // 空状态
  
  // 优惠券相关
  couponList: UserCoupon[]; // 优惠券列表
  
  // 状态选项
  statusOptions: Array<{
    value: number;
    label: string;
  }>;
}
```

## 二、加载我的优惠券列表流程

```
┌─────────────────────────────────────────────────────────────┐
│                  加载我的优惠券列表流程                       │
└─────────────────────────────────────────────────────────────┘

loadCouponList(status = 0)
  │
  ├─> 1. 显示加载状态
  │     this.setData({ loading: true, empty: false })
  │
  ├─> 2. 发起请求
  │     API: GET /api/coupon/my
  │     Params: { status: number } // 0-未使用, 1-已使用, 2-已过期
  │
  ├─> 3. 处理响应
  │     Success:
  │       - 更新 couponList
  │       - 检查是否为空数据
  │       - 设置 empty 状态
  │     Fail:
  │       - 显示错误提示
  │       - 保持原有数据不变
  │
  └─> 4. 更新页面状态
        this.setData({ 
          couponList: response,
          loading: false,
          empty: response.length === 0
        })

异常处理：
  - 网络错误：提示"网络错误，请重试"
  - 未登录：跳转到登录页
  - 服务端错误：根据错误码显示对应提示
```

## 三、优惠券状态筛选流程

```
┌─────────────────────────────────────────────────────────────┐
│                  优惠券状态筛选流程                           │
└─────────────────────────────────────────────────────────────┘

用户点击状态 Tab
  │
  ├─> 1. 更新当前选中状态
  │     this.setData({ currentStatus: status })
  │
  ├─> 2. 加载对应状态的优惠券
  │     - 调用 loadCouponList(status)
  │
  └─> 3. 页面渲染对应状态的优惠券列表

状态选项：
  - 未使用 (value: 0)
  - 已使用 (value: 1)
  - 已过期 (value: 2)
```

## 四、优惠券使用入口跳转流程

```
┌─────────────────────────────────────────────────────────────┐
│                  优惠券使用入口跳转流程                       │
└─────────────────────────────────────────────────────────────┘

用户点击"去使用"按钮
  │
  ├─> 1. 检查优惠券状态
  │     - 仅未使用状态的优惠券可点击
  │     - 已使用/已过期状态的优惠券不可点击
  │
  ├─> 2. 跳转到点单页
  │     wx.navigateTo({
  │       url: '/pages/order/order?couponId=' + couponId
  │     })
  │
  └─> 3. 点单页使用优惠券
        - 点单页接收 couponId 参数
        - 自动应用该优惠券（如果满足使用条件）
```

## 五、空优惠券状态处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                  空优惠券状态处理流程                         │
└─────────────────────────────────────────────────────────────┘

加载优惠券列表后
  │
  ├─> 1. 检查优惠券列表长度
  │     if (couponList.length === 0) {
  │       this.setData({ empty: true })
  │     }
  │
  ├─> 2. 显示空状态组件
  │     - 显示对应状态的空状态提示
  │     - 提供"去领券"入口
  │
  └─> 3. 点击"去领券"
        - 跳转到领券页面（如果存在）
        - 或跳转到首页

空状态提示文案：
  - 未使用："暂无可用优惠券，快去领取吧！"
  - 已使用："暂无已使用的优惠券"
  - 已过期："暂无已过期的优惠券"
```

## 九、异常处理策略

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示错误提示，保留原有数据 |
| 未登录 | 显示未登录状态，提供登录入口 |
| 空数据 | 显示对应状态的空状态提示 |
| 服务端错误 | 根据错误码显示对应提示 |
