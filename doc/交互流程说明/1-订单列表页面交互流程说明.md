# pages/order-list/order-list 页面交互流程说明

## 1. 页面初始化流程

1. **页面加载**
   - 小程序启动或从其他页面跳转进入订单页
   - 导航栏显示 "我的订单" 标题

2. **数据初始化**
   - 初始化页面数据：
     - `currentTab`：当前选中的订单类型，默认为 "门店订单"
     - `orderList`：订单列表数组，初始为空
     - `page`：分页页码，初始为 1
     - `pageSize`：每页订单数量，默认为 10
     - `hasMore`：是否还有更多数据，初始为 true
     - `loading`：加载状态，初始为 false
     - `refreshing`：刷新状态，初始为 false

3. **权限检查**
   - 检查用户是否已登录
   - 若未登录，提示用户登录或跳转到登录页

4. **首次数据加载**
   - 调用 `loadOrders()` 方法加载订单数据
   - 显示加载动画

## 2. 订单类型 Tab 切换流程

1. **Tab 点击事件**
   - 用户点击不同的订单类型 Tab（门店订单 | 储值订单 | 买单订单 | 券包订单）
   - 更新 `currentTab` 为选中的 Tab 类型

2. **数据重置**
   - 重置分页参数：`page` = 1，`hasMore` = true
   - 清空 `orderList` 数组
   - 显示加载动画

3. **重新加载数据**
   - 调用 `loadOrders()` 方法，传入新的订单类型
   - 根据返回数据更新订单列表

4. **UI 更新**
   - 高亮显示当前选中的 Tab
   - 显示对应类型的订单列表或空状态

## 3. 订单列表加载流程（分页）

1. **加载条件检查**
   - 检查 `loading` 状态，避免重复加载
   - 检查 `hasMore` 状态，判断是否还有更多数据

2. **API 调用**
   - 调用 `GET /api/order/list` 接口
   - 传入参数：
     - `type`：订单类型（门店订单、储值订单、买单订单、券包订单）
     - `page`：当前页码
     - `pageSize`：每页数量

3. **数据处理**
   - 若请求成功：
     - 将返回的订单数据追加到 `orderList` 数组
     - 更新 `page` 为当前页码 + 1
     - 判断返回的数据数量是否小于 `pageSize`，若小于则设置 `hasMore` = false
   - 若请求失败：
     - 显示错误提示信息

4. **UI 更新**
   - 隐藏加载动画
   - 更新订单列表显示
   - 若 `hasMore` 为 false，显示 "没有更多数据" 提示

## 4. 下拉刷新流程

1. **下拉事件触发**
   - 用户在页面顶部向下拉动
   - 触发 `onPullDownRefresh` 事件

2. **状态更新**
   - 设置 `refreshing` = true
   - 显示下拉刷新动画

3. **数据重置与加载**
   - 重置分页参数：`page` = 1，`hasMore` = true
   - 清空 `orderList` 数组
   - 调用 `loadOrders()` 方法重新加载数据

4. **刷新完成**
   - 数据加载完成后，设置 `refreshing` = false
   - 调用 `wx.stopPullDownRefresh()` 停止下拉刷新动画
   - 更新订单列表显示

## 5. 上拉加载更多流程

1. **上拉事件触发**
   - 用户滑动到页面底部
   - 触发 `onReachBottom` 事件

2. **加载条件检查**
   - 检查 `loading` 状态，避免重复加载
   - 检查 `hasMore` 状态，判断是否还有更多数据

3. **数据加载**
   - 若满足加载条件，调用 `loadOrders()` 方法加载下一页数据
   - 显示加载更多动画

4. **数据处理与 UI 更新**
   - 处理返回数据，更新 `orderList` 数组
   - 更新 `page` 和 `hasMore` 状态
   - 隐藏加载更多动画
   - 若 `hasMore` 为 false，显示 "没有更多数据" 提示

## 6. 空状态处理

1. **空状态判断**
   - 当 `orderList` 数组为空且 `loading` 为 false 时，显示空状态

2. **空状态 UI**
   - 显示空状态插图
   - 显示提示文字："您暂时还没有订单哦~"
   - 可选：显示 "去下单" 按钮，点击跳转到点单页

3. **状态切换**
   - 当加载到订单数据时，自动切换到订单列表显示
   - 当订单列表清空时，切换回空状态显示

## 7. 跳转订单详情流程

1. **订单点击事件**
   - 用户点击订单卡片或 "查看详情" 按钮

2. **参数传递**
   - 获取被点击订单的 `orderId`
   - 构造跳转参数，包含 `orderId`

3. **页面跳转**
   - 调用 `wx.navigateTo()` 方法
   - 跳转到 `pages/order-detail/order-detail` 页面
   - 传递 `orderId` 参数

4. **详情页加载**
   - 订单详情页根据 `orderId` 获取并显示订单详细信息
