# 充值中心（pages/recharge/recharge）交互流程说明

## 一、页面初始化流程

```
┌─────────────────────────────────────────────────────────────┐
│                     页面初始化流程                           │
└─────────────────────────────────────────────────────────────┘

onLoad
  │
  ├─> 1. 初始化页面数据
  │     - rechargeList: [] (充值套餐列表)
  │     - selectedPackage: null (选中的充值套餐)
  │     - userBalance: 0 (用户当前余额)
  │     - loading: false (加载状态)
  │     - error: null (错误信息)
  │
  ├─> 2. 检查登录状态
  │     - 调用 auth.checkLogin() 检查登录状态
  │     - 未登录：显示未登录提示，引导登录
  │     - 已登录：继续流程
  │
  ├─> 3. 加载用户余额
  │     API: GET /api/balance
  │     - 成功后更新 userBalance
  │     - 失败：显示错误提示
  │
  ├─> 4. 加载充值套餐列表
  │     调用 loadRechargePackages() 加载套餐
  │
  └─> 5. 页面渲染完成
```

### 数据结构设计

```typescript
interface PageData {
  // 充值套餐相关
  rechargeList: RechargePackage[];
  selectedPackage: RechargePackage | null;
  
  // 用户相关
  userBalance: number;
  isLoggedIn: boolean;
  
  // 状态相关
  loading: boolean;
  error: string | null;
  
  // 支付相关
  paymentLoading: boolean;
  paymentSuccess: boolean;
  paymentError: string | null;
}

interface RechargePackage {
  id: string;
  amount: number;      // 充值金额
  giftAmount: number;  // 赠送金额
  totalAmount: number; // 实际到账金额
  price: number;       // 实际支付价格
  description: string; // 套餐描述
  isRecommended: boolean; // 是否推荐
}
```

## 二、加载充值套餐列表流程

```
┌─────────────────────────────────────────────────────────────┐
│                   加载充值套餐列表流程                       │
└─────────────────────────────────────────────────────────────┘

loadRechargePackages()
  │
  ├─> 1. 显示加载状态
  │     this.setData({ loading: true, error: null })
  │
  ├─> 2. 发起请求
  │     API: GET /api/recharge/packages
  │     - 无需登录即可获取套餐列表
  │
  ├─> 3. 处理响应
  │     Success:
  │       - 格式化套餐数据，计算 totalAmount
  │       - 更新 rechargeList
  │       - 自动选择推荐套餐（如果有）
  │     Fail:
  │       - 显示错误提示
  │       - 保持原有数据不变
  │
  └─> 4. 更新页面状态
        this.setData({ 
          rechargeList: formattedList,
          loading: false 
        })

套餐数据格式化逻辑：
  rechargeList.map(pkg => ({
    ...pkg,
    totalAmount: pkg.amount + pkg.giftAmount
  }))
```

## 三、选择充值套餐流程

```
┌─────────────────────────────────────────────────────────────┐
│                    选择充值套餐流程                          │
└─────────────────────────────────────────────────────────────┘

用户点击充值套餐
  │
  ├─> 1. 检查登录状态
  │     - 未登录：显示未登录提示，引导登录
  │     - 已登录：继续流程
  │
  ├─> 2. 更新选中状态
  │     this.setData({ 
  │       selectedPackage: selectedPackage,
  │       paymentError: null
  │     })
  │
  ├─> 3. 视觉反馈
  │     - 高亮显示选中的套餐
  │     - 取消其他套餐的选中状态
  │
  └─> 4. 激活支付按钮
        - 支付按钮变为可点击状态
```

## 四、发起支付流程

```
┌─────────────────────────────────────────────────────────────┐
│                     发起支付流程                             │
└─────────────────────────────────────────────────────────────┘

用户点击"立即充值"按钮
  │
  ├─> 1. 检查登录状态
  │     - 未登录：显示未登录提示，引导登录
  │     - 已登录：继续流程
  │
  ├─> 2. 检查是否选择套餐
  │     - 未选择：Toast 提示"请选择充值套餐"
  │     - 已选择：继续流程
  │
  ├─> 3. 显示支付加载状态
  │     this.setData({ paymentLoading: true })
  │
  ├─> 4. 创建充值订单
  │     API: POST /api/recharge/create
  │     Params: {
  │       packageId: selectedPackage.id,
  │       amount: selectedPackage.amount,
  │       giftAmount: selectedPackage.giftAmount,
  │       price: selectedPackage.price
  │     }
  │
  ├─> 5. 调用支付接口
  │     - 使用微信支付 API
  │     - 传入订单信息
  │
  ├─> 6. 处理支付回调
  │     - 成功：进入支付成功流程
  │     - 失败：进入支付失败流程
  │     - 取消：进入支付取消流程
  │
  └─> 7. 更新支付状态
        this.setData({ paymentLoading: false })
```

## 五、支付成功流程

```
┌─────────────────────────────────────────────────────────────┐
│                     支付成功流程                             │
└─────────────────────────────────────────────────────────────┘

支付成功回调
  │
  ├─> 1. 显示支付成功提示
  │     wx.showToast({ 
  │       title: '充值成功', 
  │       icon: 'success',
  │       duration: 2000
  │     })
  │
  ├─> 2. 更新页面状态
  │     this.setData({ 
  │       paymentSuccess: true,
  │       paymentError: null
  │     })
  │
  ├─> 3. 刷新用户余额
  │     API: GET /api/balance
  │     - 更新 userBalance
  │
  ├─> 4. 提供后续操作
  │     ├─> 跳转余额明细页面
  │     │     wx.navigateTo({ url: '/pages/balance-detail/balance-detail' })
  │     │
  │     └─> 继续充值
  │           - 重置选中状态
  │           - 清除支付成功状态
  │
  └─> 5. 自动跳转（可选）
        - 3秒后自动跳转到余额明细页面
```

## 六、支付失败或取消流程

```
┌─────────────────────────────────────────────────────────────┐
│                  支付失败或取消流程                          │
└─────────────────────────────────────────────────────────────┘

支付失败或用户取消支付
  │
  ├─> 1. 显示错误提示
  │     - 失败：Toast 提示"支付失败，请重试"
  │     - 取消：Toast 提示"支付已取消"
  │
  ├─> 2. 更新页面状态
  │     this.setData({ 
  │       paymentError: errorMessage,
  │       paymentSuccess: false
  │     })
  │
  ├─> 3. 保留选中状态
  │     - 保持用户之前选择的套餐
  │     - 允许用户重新发起支付
  │
  └─> 4. 提供重试选项
        - 显示"重新支付"按钮
        - 点击后重新发起支付流程
```

## 七、未登录状态处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                    未登录状态处理流程                        │
└─────────────────────────────────────────────────────────────┘

检测到未登录状态
  │
  ├─> 1. 显示未登录提示
  │     - 显示登录按钮
  │     - 提示"请先登录"
  │
  ├─> 2. 限制操作
  │     - 禁用选择套餐功能
  │     - 禁用支付按钮
  │
  ├─> 3. 处理登录
  │     用户点击登录按钮
  │     ├─> 跳转到登录页
  │     │     wx.navigateTo({ url: '/pages/login/login' })
  │     │
  │     └─> 登录成功后返回
  │           - 重新初始化页面
  │           - 加载用户信息和余额
  │
  └─> 4. 恢复功能
        - 登录成功后，启用所有功能
        - 自动加载数据
```

## 九、关键接口汇总

| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 获取充值套餐 | /api/recharge/packages | GET | 无需登录 |
| 创建充值订单 | /api/recharge/create | POST | 需要登录 |
| 获取用户余额 | /api/balance | GET | 需要登录 |
| 检查登录状态 | /api/auth/check | GET | 检查登录状态 |

## 十、异常处理策略

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示错误提示，保留原有数据 |
| 未登录 | 显示登录提示，引导用户登录 |
| 支付失败 | 显示错误提示，允许重试 |
| 支付取消 | 显示取消提示，保持选中状态 |
| 接口限流 | 提示"操作太频繁，请稍后再试" |
| 服务端错误 | 根据错误码显示对应提示 |

## 十一、用户体验优化

1. **加载状态优化**：
   - 套餐列表加载时显示骨架屏
   - 支付过程中显示加载动画

2. **视觉反馈**：
   - 选中套餐时添加高亮效果
   - 支付成功/失败时显示明确的提示

3. **交互引导**：
   - 未登录时明确引导用户登录
   - 未选择套餐时提示用户选择

4. **性能优化**：
   - 缓存套餐列表数据
   - 防抖处理支付按钮点击

5. **安全性**：
   - 支付前再次确认订单信息
   - 防止重复提交支付请求
