# 登录页（pages/login/login）交互流程说明

## 一、页面初始化流程

```
┌─────────────────────────────────────────────────────────────┐
│                     页面初始化流程                           │
└─────────────────────────────────────────────────────────────┘

onLoad
  │
  ├─> 1. 初始化页面数据
  │     - loading: false (加载状态)
  │     - errorMessage: '' (错误信息)
  │     - loginType: 'wechat' (登录方式：wechat-微信/phone-手机号)
  │     - phone: '' (手机号)
  │     - code: '' (验证码)
  │
  ├─> 2. 检查登录状态
  │     - 调用 checkLoginStatus() 检查是否已登录
  │     - 已登录：跳转到首页或返回上一页
  │     - 未登录：继续流程
  │
  ├─> 3. 获取页面参数
  │     - 检查是否有 redirectUrl 参数
  │     - 保存到本地，登录成功后跳转
  │
  └─> 4. 页面渲染完成
```

### 数据结构设计

```typescript
interface PageData {
  // 状态相关
  loading: boolean;         // 加载状态
  errorMessage: string;     // 错误信息
  
  // 登录方式
  loginType: 'wechat' | 'phone';  // 登录方式
  
  // 手机号登录相关
  phone: string;            // 手机号
  code: string;             // 验证码
  countDown: number;        // 验证码倒计时
  
  // 其他
  redirectUrl: string;      // 登录成功后跳转地址
}
```

## 二、微信登录流程

```
┌─────────────────────────────────────────────────────────────┐
│                     微信登录流程                             │
└─────────────────────────────────────────────────────────────┘

用户点击"微信登录"按钮
  │
  ├─> 1. 显示加载状态
  │     this.setData({ loading: true })
  │
  ├─> 2. 调用 wx.login() 获取 code
  │     wx.login({
  │       success: (res) => {
  │         if (res.code) {
  │           // 继续登录流程
  │         } else {
  │           // 显示错误提示
  │         }
  │       },
  │       fail: (err) => {
  │         // 显示错误提示
  │       }
  │     })
  │
  ├─> 3. 调用登录接口
  │     API: POST /api/auth/login
  │     Params: { code: string }
  │
  ├─> 4. 处理登录响应
  │     Success:
  │       - 保存 token 到本地
  │       - 保存用户信息到本地
  │       - 跳转到首页或 redirectUrl
  │     Fail:
  │       - 显示错误提示
  │       - 隐藏加载状态
  │
  └─> 5. 完成登录流程
```

### 微信登录核心代码

```typescript
async handleWechatLogin() {
  try {
    this.setData({ loading: true, errorMessage: '' });
    
    // 获取微信登录 code
    const code = await new Promise<string>((resolve, reject) => {
      wx.login({
        success: (res) => {
          if (res.code) {
            resolve(res.code);
          } else {
            reject(new Error('获取登录凭证失败'));
          }
        },
        fail: (err) => reject(err)
      });
    });
    
    // 调用登录接口
    const response = await login(code);
    
    // 保存登录信息
    wx.setStorageSync('token', response.token);
    wx.setStorageSync('userInfo', response.userInfo);
    
    // 跳转到目标页面
    this.redirectAfterLogin();
  } catch (error) {
    this.setData({ 
      errorMessage: error.message || '登录失败，请重试',
      loading: false 
    });
  }
}
```

### 获取手机号核心代码

```typescript
async handleGetPhoneNumber(e: any) {
  try {
    this.setData({ loading: true, errorMessage: '' });
    
    // 获取微信登录 code
    const code = await new Promise<string>((resolve, reject) => {
      wx.login({
        success: (res) => {
          if (res.code) {
            resolve(res.code);
          } else {
            reject(new Error('获取登录凭证失败'));
          }
        },
        fail: (err) => reject(err)
      });
    });
    
    // 获取手机号
    const { encryptedData, iv } = e.detail;
    const phoneResponse = await getPhoneNumber(encryptedData, iv);
    
    // 调用登录接口
    const response = await login(code, phoneResponse.phone);
    
    // 保存登录信息
    wx.setStorageSync('token', response.token);
    wx.setStorageSync('userInfo', response.userInfo);
    
    // 跳转到目标页面
    this.redirectAfterLogin();
  } catch (error) {
    this.setData({ 
      errorMessage: error.message || '登录失败，请重试',
      loading: false 
    });
  }
}
```

## 四、登录成功后的返回流程

```
┌─────────────────────────────────────────────────────────────┐
│                   登录成功后的返回流程                        │
└─────────────────────────────────────────────────────────────┘

登录成功
  │
  ├─> 1. 保存登录状态
  │     - token: 保存到本地存储
  │     - userInfo: 保存到本地存储
  │
  ├─> 2. 检查 redirectUrl
  │     ├─> 有 redirectUrl
  │     │     - 跳转到指定地址
  │     │     - 清除 redirectUrl 参数
  │     │
  │     └─> 无 redirectUrl
  │           - 跳转到首页
  │           - 或返回上一页
  │
  ├─> 3. 显示成功提示
  │     wx.showToast({ title: '登录成功' })
  │
  └─> 4. 完成跳转
```

### 跳转逻辑代码

```typescript
redirectAfterLogin() {
  const { redirectUrl } = this.data;
  
  if (redirectUrl) {
    // 跳转到指定地址
    wx.redirectTo({
      url: redirectUrl,
      success: () => {
        wx.showToast({ title: '登录成功' });
      }
    });
  } else {
    // 跳转到首页或返回上一页
    wx.navigateBack({
      delta: 1,
      success: () => {
        wx.showToast({ title: '登录成功' });
      }
    });
  }
}
```

## 五、登录失败与错误提示流程

```
┌─────────────────────────────────────────────────────────────┐
│                  登录失败与错误提示流程                       │
└─────────────────────────────────────────────────────────────┘

登录失败
  │
  ├─> 1. 隐藏加载状态
  │     this.setData({ loading: false })
  │
  ├─> 2. 显示错误信息
  │     ├─> 网络错误
  │     │     - 提示"网络错误，请检查网络连接"
  │     │
  │     ├─> 微信登录失败
  │     │     - 提示"微信登录失败，请重试"
  │     │
  │     ├─> 手机号获取失败
  │     │     - 提示"获取手机号失败，请重试"
  │     │
  │     ├─> 服务端错误
  │     │     - 根据错误码显示对应提示
  │     │     - 如："账号或密码错误"
  │     │
  │     └─> 其他错误
  │           - 提示"登录失败，请稍后重试"
  │
  ├─> 3. 记录错误日志
  │     console.error('登录失败:', error)
  │
  └─> 4. 允许用户重新尝试
      - 保持页面状态
      - 允许用户再次点击登录按钮
```

### 错误处理代码

```typescript
showError(message: string) {
  this.setData({ 
    errorMessage: message,
    loading: false 
  });
  
  // 3秒后自动清除错误信息
  setTimeout(() => {
    this.setData({ errorMessage: '' });
  }, 3000);
}
```

## 六、已登录状态下的处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                  已登录状态下的处理流程                       │
└─────────────────────────────────────────────────────────────┘

页面初始化时
  │
  ├─> 1. 检查本地存储
  │     - 检查是否存在 token
  │     - 检查 token 是否有效
  │
  ├─> 2. 验证登录状态
  │     ├─> 已登录
  │     │     - 跳转到首页或 redirectUrl
  │     │     - 显示"已登录"提示
  │     │
  │     └─> 未登录或 token 无效
  │           - 继续显示登录页面
  │
  └─> 3. 完成状态检查
```

### 登录状态检查代码

```typescript
checkLoginStatus() {
  const token = wx.getStorageSync('token');
  
  if (token) {
    // 跳转到目标页面
    this.redirectAfterLogin();
    return true;
  }
  
  return false;
}
```

## 十、页面交互细节

### 1. 加载状态
- 登录按钮点击后显示加载中状态
- 禁用登录按钮，防止重复点击
- 加载完成后恢复按钮状态

### 2. 错误提示
- 错误信息显示在页面顶部
- 3秒后自动消失
- 支持用户手动关闭

### 3. 登录方式切换
- 提供微信登录和手机号登录两种方式
- 支持切换登录方式
- 保存用户登录方式偏好

### 4. 视觉反馈
- 登录成功/失败都有相应的 Toast 提示
- 按钮状态变化提供视觉反馈
- 加载过程显示 loading 动画

### 5. 安全性
- 敏感信息（如 token）加密存储
- 登录凭证（code）及时使用，避免过期
- 防止登录凭证泄露

## 十三、总结

登录页面是用户进入应用的第一道门槛，需要提供简洁、流畅的登录体验。通过微信登录和手机号登录两种方式，满足不同用户的需求。同时，完善的错误处理和状态管理，确保用户在各种情况下都能得到清晰的反馈。

本交互流程说明涵盖了登录页面的完整流程，包括页面初始化、登录流程、错误处理等各个环节，为开发人员提供了详细的实现指导。
