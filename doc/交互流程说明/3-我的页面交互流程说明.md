# 我的页面（pages/user/user）交互流程说明

## 一、页面初始化流程

```
┌─────────────────────────────────────────────────────────────┐
│                     页面初始化流程                           │
└─────────────────────────────────────────────────────────────┘

onLoad
  │
  ├─> 1. 初始化页面数据
  │     - userInfo: null (用户信息)
  │     - isLoggedIn: false (登录状态)
  │     - balance: 0 (余额)
  │     - couponCount: 0 (优惠券数量)
  │     - orderCount: { pending: 0, shipping: 0, completed: 0 } (订单数量)
  │
  ├─> 2. 检查登录状态
  │     - 从本地存储获取 token
  │     - token 存在：设置 isLoggedIn: true，调用 loadUserInfo()
  │     - token 不存在：保持未登录状态，显示登录入口
  │
  ├─> 3. 页面渲染完成
  │     - 根据登录状态显示不同 UI
  │     - 未登录：显示登录按钮
  │     - 已登录：显示用户信息和功能列表
  │
  └─> 4. 加载用户相关数据（已登录状态）
        - 调用 loadUserInfo() 获取用户信息
        - 调用 loadBalance() 获取余额
        - 调用 loadCouponCount() 获取优惠券数量
        - 调用 loadOrderCount() 获取订单数量
```

### 数据结构设计

```typescript
interface PageData {
  // 用户信息
  userInfo: UserInfo | null;
  isLoggedIn: boolean;
  loading: boolean;
  
  // 统计信息
  balance: number;
  couponCount: number;
  orderCount: {
    pending: number;     // 待付款
    shipping: number;    // 配送中
    completed: number;   // 已完成
  };
  
  // 页面状态
  showLogoutConfirm: boolean;
}
```

## 二、加载用户信息流程

```
┌─────────────────────────────────────────────────────────────┐
│                   加载用户信息流程                           │
└─────────────────────────────────────────────────────────────┘

loadUserInfo()
  │
  ├─> 1. 显示加载状态
  │     this.setData({ loading: true })
  │
  ├─> 2. 发起请求
  │     API: GET /api/user/info
  │     - 需要携带 token
  │
  ├─> 3. 处理响应
  │     Success:
  │       - 更新用户信息
  │       - 设置 isLoggedIn: true
  │       - 调用 loadUserStats() 加载统计数据
  │     Fail:
  │       - 401 错误：清除本地 token，设置 isLoggedIn: false
  │       - 其他错误：显示错误提示
  │
  └─> 4. 更新页面状态
        this.setData({ 
          userInfo: response,
          isLoggedIn: true,
          loading: false 
        })

loadUserStats()
  │
  ├─> 1. 并行加载统计数据
  │     - loadBalance()
  │     - loadCouponCount()
  │     - loadOrderCount()
  │
  └─> 2. 更新页面显示
```

## 三、未登录状态处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                   未登录状态处理流程                         │
└─────────────────────────────────────────────────────────────┘

未登录状态下用户操作
  │
  ├─> 1. 点击登录按钮
  │     - 调用 wx.login() 获取 code
  │     - 调用 login API 进行登录
  │     API: POST /api/auth/login
  │     Params: { code }
  │
  ├─> 2. 登录成功
  │     - 保存 token 到本地存储
  │     - 设置 isLoggedIn: true
  │     - 调用 loadUserInfo() 加载用户信息
  │     - 显示登录成功提示
  │
  ├─> 3. 登录失败
  │     - 显示错误提示
  │     - 保持未登录状态
  │
  └─> 4. 未登录时点击其他功能
        - 弹出登录提示
        - 引导用户登录
```

## 九、退出登录流程

```
┌─────────────────────────────────────────────────────────────┐
│                   退出登录流程                               │
└─────────────────────────────────────────────────────────────┘

  │

  │
  └─> 4. 退出成功
        - 显示退出成功提示

```

## 十一、关键接口汇总

| 功能      | 接口                    | 方法   | 说明            |
| ------- | --------------------- | ---- | ------------- |
| 获取用户信息  | /api/user/info        | GET  | 需要登录          |
| 更新用户信息  | /api/user/info        | PUT  | 需要登录          |
| 微信登录    | /api/auth/login       | POST | 无需登录，使用微信code |
| 获取手机号   | /api/auth/phone       | POST | 需要登录          |
| 获取地址列表  | /api/address/list     | GET  | 需要登录          |
| 获取我的优惠券 | /api/coupon/my        | GET  | 需要登录，支持状态筛选   |
| 获取余额明细  | /api/balance/records  | GET  | 需要登录，支持分页     |
| 余额充值    | /api/balance/recharge | POST | 需要登录          |

## 十二、异常处理策略

| 场景    | 处理方式             |
| ----- | ---------------- |
| 网络错误  | 显示错误提示，保留原有数据    |
| 未登录   | 跳转登录流程，登录成功后返回   |
| 登录失效  | 清除本地token，显示登录入口 |
| 接口限流  | 提示"操作太频繁，请稍后再试"  |
| 服务端错误 | 根据错误码显示对应提示      |
| 空数据   | 显示空状态组件          |

## 十三、页面交互优化

1. **加载状态管理**
   - 页面初始化时显示骨架屏
   - 数据加载时显示加载动画
   - 加载失败时显示错误提示和重试按钮
2. **用户体验优化**
   - 未登录时点击功能项自动引导登录
   - 登录成功后自动返回原页面
   - 退出登录时使用确认弹窗防止误操作
   - 数据更新时使用 Toast 提示
3. **性能优化**
   - 并行加载用户统计数据
   - 使用本地缓存减少重复请求
   - 延迟加载非关键数据
4. **安全处理**
   - 退出登录时清除所有用户相关缓存
   - 敏感操作使用确认机制
   - 定期检查登录状态有效性
