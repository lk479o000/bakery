# 点单页（pages/order/order）交互流程说明

## 一、页面初始化流程

```
┌─────────────────────────────────────────────────────────────┐
│                     页面初始化流程                           │
└─────────────────────────────────────────────────────────────┘

onLoad
  │
  ├─> 1. 初始化页面数据
  │     - currentCategoryId: '' (当前选中分类ID)
  │     - categoryList: [] (分类列表)
 │     - productList: [] (商品列表)
  │     - cartList: [] (购物车列表)
  │     - orderType: 'pickup' (订单类型：pickup-自提/delivery-外卖)
  │     - cartTotal: { count: 0, amount: 0 } (购物车统计)
  │
  ├─> 2. 加载分类列表
  │     API: GET /api/category/list
  │     - 成功后默认选中第一个分类
  │     - 调用 loadProductList() 加载商品
  │
  ├─> 3. 加载购物车数据
  │     API: GET /api/cart/list
  │     - 更新购物车列表
  │     - 计算购物车统计
  │
  └─> 4. 页面渲染完成
```

### 数据结构设计

```typescript
interface PageData {
  // 分类相关
  categoryList: Category[];
  currentCategoryId: string;
  
  // 商品相关
  productList: Product[];
  loading: boolean;
  
  // 购物车相关
  cartList: CartItem[];
  cartTotal: {
    count: number;
    amount: number;
  };
  
  // 订单类型
  orderType: 'pickup' | 'delivery';
  
  // 购物车栏展开状态
  cartPanelVisible: boolean;
}
```

## 二、分类切换流程

```
┌─────────────────────────────────────────────────────────────┐
│                     分类切换流程                             │
└─────────────────────────────────────────────────────────────┘

用户点击分类 Tab
  │
  ├─> 1. 更新当前选中分类ID
  │     this.setData({ currentCategoryId: categoryId })
  │
  ├─> 2. 显示加载状态
  │     this.setData({ loading: true })
  │
  ├─> 3. 加载该分类商品
  │     API: GET /api/product/list
  │     Params: { categoryId, page: 1, pageSize: 20 }
  │
  ├─> 4. 更新商品列表
  │     this.setData({ 
  │       productList: response.list,
  │       loading: false 
  │     })
  │
  └─> 5. 页面渲染新分类商品

异常处理：
  - 网络错误：显示错误提示，保留原分类数据
  - 空数据：显示空状态组件
```

## 三、商品列表加载流程

```
┌─────────────────────────────────────────────────────────────┐
│                   商品列表加载流程                           │
└─────────────────────────────────────────────────────────────┘

loadProductList(categoryId)
  │
  ├─> 1. 参数校验
  │     - 检查 categoryId 是否有效
  │     - 设置默认分页参数
  │
  ├─> 2. 发起请求
  │     API: GET /api/product/list
  │     Params: {
  │       categoryId: string,  // 分类ID
  │       page: number,         // 页码，默认1
  │       pageSize: number      // 每页数量，默认20
  │     }
  │
  ├─> 3. 处理响应
  │     Success:
  │       - 更新 productList
  │       - 合并购物车数量到商品列表
  │     Fail:
  │       - 显示错误提示
  │       - 保持原有数据不变
  │
  └─> 4. 更新页面状态
        this.setData({ 
          productList: mergedList,
          loading: false 
        })

商品列表与购物车数据合并逻辑：
  productList.map(product => {
    const cartItem = cartList.find(item => item.productId === product.id);
    return {
      ...product,
      cartQuantity: cartItem ? cartItem.quantity : 0
    };
  })
```

## 四、加入购物车流程

```
┌─────────────────────────────────────────────────────────────┐
│                    加入购物车流程                            │
└─────────────────────────────────────────────────────────────┘

用户点击"加入购物车"或"+"按钮
  │
  ├─> 1. 检查登录状态
  │     - 未登录：跳转登录页
  │     - 已登录：继续流程
  │
  ├─> 2. 检查商品库存
  │     - 库存不足：提示"库存不足"
  │     - 库存充足：继续流程
  │
  ├─> 3. 查找购物车中是否已有该商品
  │     const existingItem = cartList.find(
  │       item => item.productId === productId
  │     )
  │
  ├─> 4. 判断操作类型
  │     ├─> 4.1 购物车中已存在
  │     │       API: PUT /api/cart/update
  │     │       Params: {
  │     │         cartId: existingItem.id,
  │     │         quantity: existingItem.quantity + 1
  │     │       }
 │     │
  │     └─> 4.2 购物车中不存在
  │             API: POST /api/cart/add
  │             Params: {
  │               productId: string,
  │               quantity: 1
  │             }
  │
  ├─> 5. 更新本地购物车数据
  │     - 重新调用 loadCartList() 刷新购物车
  │     - 或者本地更新后同步服务端
  │
  ├─> 6. 更新商品列表中的购物车数量
  │     - 找到对应商品，更新 cartQuantity
  │
  ├─> 7. 显示成功提示
  │     wx.showToast({ title: '已加入购物车' })
  │
  └─> 8. 更新购物车栏显示

异常处理：
  - 网络错误：提示"网络错误，请重试"
  - 服务端错误：根据错误码显示对应提示
```

## 五、购物车栏更新流程

```
┌─────────────────────────────────────────────────────────────┐
│                   购物车栏更新流程                           │
└─────────────────────────────────────────────────────────────┘

触发时机：
  1. 页面初始化时
  2. 加入购物车时
  3. 购物车数量变更时
  4. 删除购物车商品时
  5. 清空购物车时

updateCartBar()
  │
  ├─> 1. 计算购物车统计
  │     const count = cartList.reduce(
  │       (sum, item) => sum + item.quantity, 
  │       0
  │     )
  │     const amount = cartList
  │       .filter(item => item.selected)
  │       .reduce((sum, item) => sum + item.price * item.quantity, 0)
  │
  ├─> 2. 更新购物车栏显示
  │     this.setData({
  │       cartTotal: { count, amount }
  │     })
  │
  └─> 3. 更新商品列表中的购物车数量标记
        this.updateProductCartQuantity()

购物车栏交互：
  ├─> 点击购物车图标
  │     - 展开购物车面板 (cartPanelVisible: true)
  │     - 显示购物车商品列表
  │     - 支持修改数量、删除商品
  │
  ├─> 点击"去结算"按钮
  │     - 检查是否已选商品
  │     - 跳转到订单确认页
  │
  └─> 点击遮罩层
        - 关闭购物车面板 (cartPanelVisible: false)

购物车面板功能：
  ├─> 数量增加/减少
  │     API: PUT /api/cart/update
  │     - 数量为0时自动删除
  │
  ├─> 删除商品
  │     API: DELETE /api/cart/delete
  │
  └─> 清空购物车
        API: DELETE /api/cart/clear
```

## 六、自提/外卖切换流程

```
┌─────────────────────────────────────────────────────────────┐
│                  自提/外卖切换流程                           │
└─────────────────────────────────────────────────────────────┘

用户点击自提/外卖切换按钮
  │
  ├─> 1. 更新订单类型
  │     this.setData({ 
  │       orderType: type // 'pickup' | 'delivery'
  │     })
  │
  ├─> 2. 保存用户偏好到本地
  │     wx.setStorageSync('orderType', type)
  │
  ├─> 3. 根据类型显示不同提示
  │     ├─> 自提模式
  │     │     - 显示门店地址
  │     │     - 显示营业时间
  │     │     - 提示"下单后到店自取"
  │     │
  │     └─> 外卖模式
  │           - 检查是否已设置收货地址
  │           - 未设置：提示设置地址
  │           - 已设置：显示默认地址
  │           - 提示"配送到家"
  │
  └─> 4. 更新结算按钮文案
        - 自提："选好了"
        - 外卖："选好了"

页面初始化时读取偏好：
  onLoad() {
    const savedType = wx.getStorageSync('orderType') || 'pickup';
    this.setData({ orderType: savedType });
  }
```

## 八、关键接口汇总

| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 获取分类 | /api/category/list | GET | 无需登录 |
| 获取商品 | /api/product/list | GET | 无需登录，支持分页 |
| 获取购物车 | /api/cart/list | GET | 需要登录 |
| 添加购物车 | /api/cart/add | POST | 需要登录 |
| 更新购物车 | /api/cart/update | PUT | 需要登录 |
| 删除购物车 | /api/cart/delete | DELETE | 需要登录 |
| 清空购物车 | /api/cart/clear | DELETE | 需要登录 |

## 九、异常处理策略

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示错误提示，保留原有数据 |
| 未登录 | 跳转登录页，登录成功后返回 |
| 库存不足 | Toast 提示"库存不足" |
| 接口限流 | 提示"操作太频繁，请稍后再试" |
| 服务端错误 | 根据错误码显示对应提示 |
