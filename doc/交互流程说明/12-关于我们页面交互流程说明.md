# 关于我们（pages/about/about）交互流程说明

## 一、页面初始化流程

```
┌─────────────────────────────────────────────────────────────┐
│                     页面初始化流程                           │
└─────────────────────────────────────────────────────────────┘

onLoad
  │
  ├─> 1. 初始化页面数据
  │     - brandInfo: null (品牌信息)
  │     - storeList: [] (门店列表)
  │     - loading: true (加载状态)
  │     - activeTab: 'brand' (当前激活标签：brand-品牌介绍/store-门店信息)
  │     - showMap: false (地图显示状态)
  │
  ├─> 2. 加载品牌介绍内容
  │     - 调用 loadBrandInfo() 方法
  │
  ├─> 3. 加载门店信息
  │     - 调用 loadStoreList() 方法
  │
  └─> 4. 页面渲染完成
      - 设置 loading: false
      - 显示页面内容
```

### 数据结构设计

```typescript
interface PageData {
  // 品牌信息
  brandInfo: {
    name: string;           // 品牌名称
    logo: string;           // 品牌 logo
    description: string;    // 品牌描述
    history: string;        // 品牌历史
    mission: string;        // 品牌使命
    vision: string;         // 品牌愿景
    values: string[];       // 品牌价值观
  } | null;
  
  // 门店信息
  storeList: {
    id: string;             // 门店 ID
    name: string;           // 门店名称
    address: string;        // 门店地址
    phone: string;          // 门店电话
    hours: string;          // 营业时间
    latitude: number;       // 纬度
    longitude: number;      // 经度
    image: string;          // 门店图片
  }[];
  
  // 页面状态
  loading: boolean;         // 加载状态
  activeTab: 'brand' | 'store';  // 当前激活标签
  showMap: boolean;         // 地图显示状态
  selectedStore: any;       // 选中的门店信息（用于地图展示）
}
```

## 二、加载品牌介绍内容流程

```
┌─────────────────────────────────────────────────────────────┐
│                 加载品牌介绍内容流程                         │
└─────────────────────────────────────────────────────────────┘

loadBrandInfo()
  │
  ├─> 1. 检查本地缓存
  │     - 调用 wx.getStorageSync('brandInfo')
  │     - 若存在缓存且未过期，直接使用缓存数据
  │
  ├─> 2. 缓存不存在或已过期，从服务端拉取
  │     API: GET /api/about/brand
  │
  ├─> 3. 处理响应
  │     Success:
  │       - 更新 brandInfo 数据
  │       - 缓存数据到本地（设置过期时间）
  │     Fail:
  │       - 使用默认品牌信息
  │       - 显示错误提示
  │
  └─> 4. 更新页面状态
        this.setData({ 
          brandInfo: response.data,
          loading: false 
        })

缓存策略：
  - 缓存键：'brandInfo'
  - 缓存时间：24小时
  - 缓存结构：{ data: brandInfo, expire: timestamp }
```

## 四、跳转地图或导航流程

```
┌─────────────────────────────────────────────────────────────┐
│                  跳转地图或导航流程                           │
└─────────────────────────────────────────────────────────────┘

用户点击门店的"查看地图"或"导航"按钮
  │
  ├─> 1. 检查用户位置权限
  │     - 调用 wx.getLocation() 获取位置权限
  │     - 若未授权，提示用户开启位置权限
  │
  ├─> 2. 打开地图选择器
  │     wx.chooseLocation({
  │       latitude: store.latitude,
  │       longitude: store.longitude,
  │       name: store.name,
  │       address: store.address,
  │       success: res => {
  │         // 处理选择结果
  │       }
  │     })
  │
  ├─> 3. 选择导航方式
  │     ├─> 方案一：使用微信内置地图
  │     │     wx.openLocation({
  │     │       latitude: store.latitude,
  │     │       longitude: store.longitude,
  │     │       name: store.name,
  │     │       address: store.address,
  │     │       scale: 18
  │     │     })
  │     │
  │     └─> 方案二：跳转第三方地图应用
  │           - 生成导航链接
  │           - 使用 wx.openScheme() 打开
  │
  └─> 4. 导航完成
        - 记录导航事件（可选）
        - 关闭地图界面

位置权限处理：
  - 首次请求：显示权限申请弹窗
  - 拒绝授权：引导用户在设置中开启权限
  - 授权成功：获取用户位置，计算距离
```

## 五、静态内容更新策略说明

```
┌─────────────────────────────────────────────────────────────┐
│                 静态内容更新策略说明                         │
└─────────────────────────────────────────────────────────────┘

1. 缓存策略
   ├─> 本地缓存
   │     - 品牌信息：缓存24小时
   │     - 门店信息：缓存24小时
   │     - 缓存键：'brandInfo'、'storeList'
   │
   ├─> 缓存结构
   │     {
   │       data: {...},      // 数据内容
   │       expire: 1234567890,  // 过期时间戳
   │       version: '1.0'    // 数据版本号
   │     }
   │
   └─> 缓存更新触发条件
         - 缓存过期
         - 页面下拉刷新
         - 用户手动点击刷新按钮

2. 服务端拉取策略
   ├─> 接口设计
   │     - GET /api/about/brand：获取品牌信息
   │     - GET /api/about/stores：获取门店列表
   │     - 支持版本号参数，用于增量更新
   │
   ├─> 响应格式
   │     {
   │       code: 0,
   │       message: 'success',
   │       data: {...},       // 数据内容
   │       version: '1.0',    // 数据版本号
   │       timestamp: 1234567890  // 服务器时间戳
   │     }
   │
   └─> 失败处理
         - 网络错误：使用缓存数据
         - 服务端错误：使用默认数据
         - 无网络：使用缓存数据或默认数据

3. 数据版本管理
   ├─> 版本号机制
   │     - 服务端维护数据版本号
   │     - 客户端缓存版本号
   │     - 版本不一致时更新数据
   │
   └─> 强制更新
         - 小程序版本更新时
         - 检测到数据版本变化时
```

## 七、关键接口汇总

| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 获取品牌信息 | /api/about/brand | GET | 无需登录，支持缓存 |
| 获取门店列表 | /api/about/stores | GET | 无需登录，支持缓存 |
| 获取用户位置 | wx.getLocation | 微信API | 需要位置权限 |
| 选择位置 | wx.chooseLocation | 微信API | 打开地图选择器 |
| 打开地图 | wx.openLocation | 微信API | 打开内置地图 |
| 打开第三方应用 | wx.openScheme | 微信API | 跳转到第三方地图应用 |

## 八、异常处理策略

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示错误提示，使用缓存数据 |
| 服务端错误 | 显示错误提示，使用默认数据 |
| 无缓存数据 | 显示默认数据，提示网络异常 |
| 位置权限拒绝 | 提示用户开启权限，引导至设置页面 |
| 地图打开失败 | 提示"无法打开地图"，提供其他联系方式 |
| 数据加载超时 | 显示加载失败提示，提供重试按钮 |

## 九、性能优化策略

| 优化点 | 策略 |
|--------|------|
| 首屏加载 | 使用缓存数据，减少网络请求 |
| 数据更新 | 增量更新，只更新变化的数据 |
| 图片加载 | 图片懒加载，使用合适的图片尺寸 |
| 页面渲染 | 虚拟列表，避免一次性渲染大量门店 |
| 定位优化 | 只在需要时获取位置信息 |
| 缓存管理 | 定期清理过期缓存，避免占用过多存储空间 |
