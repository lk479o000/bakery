# 余额明细页面交互流程说明

## 1. 页面初始化流程

### 1.1 页面加载
- 用户进入 `pages/balance-record/balance-record` 页面
- 页面初始化，设置默认状态：
  - 余额明细列表为空数组
  - 分页参数：`page=1`, `pageSize=10`
  - 加载状态：`isLoading=false`
  - 下拉刷新状态：`isRefreshing=false`
  - 上拉加载状态：`isLoadingMore=false`
  - 无更多数据状态：`noMoreData=false`
  - 空状态：`isEmpty=false`
  - 未登录状态：`isLogin=false`

### 1.2 登录状态检查
- 检查用户登录状态（通过本地存储的 token）
- 如果未登录：
  - 显示未登录提示
  - 提供登录按钮
- 如果已登录：
  - 执行余额明细列表加载流程

## 2. 加载余额明细列表流程

### 2.1 发起请求
- 调用 `getBalanceRecords` 接口，参数：`{ page: 1, pageSize: 10 }`
- 设置 `isLoading=true`，显示加载中提示

### 2.2 响应处理
- **成功响应**：
  - 解析返回数据，更新余额明细列表
  - 检查返回的列表长度：
    - 如果列表长度为 0，设置 `isEmpty=true`，显示空状态
    - 如果列表长度小于 `pageSize`，设置 `noMoreData=true`
  - 更新 `total` 数据
  - 设置 `isLoading=false`，隐藏加载中提示
- **失败响应**：
  - 显示错误提示（如网络错误、服务器错误等）
  - 设置 `isLoading=false`，隐藏加载中提示

## 3. 分页加载流程（上拉加载更多）

### 3.1 触发条件
- 用户滑动页面到底部
- 满足条件：
  - `isLoadingMore=false`（不在加载中）
  - `noMoreData=false`（还有数据）
  - `isLoading=false`（不在初始加载中）

### 3.2 执行加载
- 增加 `page` 参数（`page++`）
- 设置 `isLoadingMore=true`，显示上拉加载提示
- 调用 `getBalanceRecords` 接口，参数：`{ page, pageSize: 10 }`

### 3.3 响应处理
- **成功响应**：
  - 解析返回数据，将新数据追加到余额明细列表
  - 检查返回的列表长度：
    - 如果列表长度小于 `pageSize`，设置 `noMoreData=true`
  - 更新 `total` 数据
  - 设置 `isLoadingMore=false`，隐藏上拉加载提示
- **失败响应**：
  - 显示错误提示
  - 设置 `isLoadingMore=false`，隐藏上拉加载提示
  - 回滚 `page` 参数（`page--`）

## 4. 下拉刷新流程

### 4.1 触发条件
- 用户下拉页面，触发下拉刷新

### 4.2 执行刷新
- 重置分页参数：`page=1`
- 重置状态：
  - `isRefreshing=true`
  - `noMoreData=false`
  - `isEmpty=false`
- 调用 `getBalanceRecords` 接口，参数：`{ page: 1, pageSize: 10 }`

### 4.3 响应处理
- **成功响应**：
  - 解析返回数据，替换余额明细列表
  - 检查返回的列表长度：
    - 如果列表长度为 0，设置 `isEmpty=true`，显示空状态
    - 如果列表长度小于 `pageSize`，设置 `noMoreData=true`
  - 更新 `total` 数据
  - 停止下拉刷新动画
  - 设置 `isRefreshing=false`
- **失败响应**：
  - 显示错误提示
  - 停止下拉刷新动画
  - 设置 `isRefreshing=false`

## 5. 空明细状态处理流程

### 5.1 触发条件
- 加载余额明细列表后，返回的列表长度为 0

### 5.2 显示处理
- 隐藏余额明细列表
- 显示空状态提示：
  - 图标：如钱包或账单图标
  - 文字："暂无余额明细"
  - 可能的操作：返回首页或充值按钮

## 6. 未登录状态处理流程

### 6.1 触发条件
- 页面初始化时检查到用户未登录

### 6.2 显示处理
- 隐藏余额明细列表
- 显示未登录提示：
  - 图标：如用户头像图标
  - 文字："请登录查看余额明细"
  - 操作按钮："去登录"

### 6.3 登录后处理
- 用户点击"去登录"按钮，跳转到登录页面
- 登录成功后，返回余额明细页面
- 自动执行余额明细列表加载流程

## 7. 数据结构

### 7.1 余额记录数据结构
```typescript
interface BalanceRecord {
  id: string;          // 记录ID
  type: string;        // 类型（如：recharge-充值，consume-消费）
  typeText: string;    // 类型文本（如："充值"，"消费"）
  amount: number;      // 金额
  balance: number;     // 余额
  remark: string;      // 备注
  createTime: string;  // 创建时间
}
```

### 7.2 余额记录列表响应结构
```typescript
interface BalanceRecordResponse {
  list: BalanceRecord[];  // 余额记录列表
  total: number;          // 总记录数
}
```

## 8. 接口调用

### 8.1 获取余额明细
- **接口**：`GET /api/balance/records`
- **参数**：
  - `page`：页码（默认 1）
  - `pageSize`：每页数量（默认 10）
- **返回**：`BalanceRecordResponse`

## 9. 异常处理

### 9.1 网络错误
- 显示网络错误提示
- 提供重试按钮

### 9.2 服务器错误
- 显示服务器错误提示
- 提供重试按钮

### 9.3 权限错误（未登录）
- 跳转到登录页面
- 登录成功后返回原页面

## 10. 用户体验优化

### 10.1 加载状态
- 初始加载时显示加载动画
- 上拉加载时显示加载提示
- 下拉刷新时显示刷新动画

### 10.2 错误提示
- 错误信息清晰明确
- 提供重试操作

### 10.3 空状态
- 空状态提示友好
- 提供相关操作建议

### 10.4 未登录状态
- 未登录提示清晰
- 提供便捷的登录入口
