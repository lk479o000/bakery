
# Vue Vben Admin 项目使用指南

## 概述

本项目基于 vue-vben-admin 模板构建。为了避免将模板代码提交到团队仓库，我们只保留了自定义业务代码。其他同事获取项目后，需要按照以下步骤从官方仓库拉取模板并应用团队代码。

## 前置要求

- Node.js: >= 22.22.0
- pnpm: >= 10.0.0

## 完整部署步骤

### 1. 克隆官方模板仓库

```bash
# 创建项目目录并进入
cd vue-vben-admin

# 克隆官方模板（使用 --depth=1 只获取最新版本）
git clone --depth=1 https://github.com/vbenjs/vue-vben-admin.git .
```

### 2. 添加团队仓库作为远程

```bash
# 添加团队仓库远程地址
git remote add bakery <团队仓库地址>

# 查看远程仓库配置
git remote -v
```

### 3. 拉取团队自定义代码

```bash
# 拉取团队分支代码
git fetch bakery

# 合并团队代码到当前分支（假设团队分支为 main）
git merge bakery/main --allow-unrelated-histories
```

### 4. 安装依赖

```bash
# 安装项目依赖
pnpm install
```

### 5. 启动开发服务器

```bash
# 启动面包店业务模块（推荐）
pnpm dev:naive

# 或者使用 filter 方式直接运行：
pnpm --filter @vben/web-naive dev

# 启动其他模块：
# pnpm dev:antd    - Ant Design 版本
# pnpm --filter @vben/web-antd dev
# pnpm dev:ele     - Element Plus 版本
# pnpm --filter @vben/web-ele dev
# pnpm dev:tdesign - TDesign 版本
# pnpm --filter @vben/web-tdesign dev
# pnpm dev:play    - 演示版本
# pnpm --filter @vben/playground dev
```

### 6. 构建生产版本

```bash
# 构建面包店业务模块
pnpm build:naive

# 或者使用 filter 方式直接构建：
pnpm --filter @vben/web-naive build

# 构建其他模块：
# pnpm build:antd
# pnpm --filter @vben/web-antd build
# pnpm build:ele
# pnpm --filter @vben/web-ele build
# pnpm build:tdesign
# pnpm --filter @vben/web-tdesign build
# pnpm build:play
# pnpm --filter @vben/playground build
```

## 项目结构说明

```
vue-vben-admin/
├── apps/                    # 团队自定义代码（已提交）
│   ├── backend-mock/        # 后端 Mock API
│   │   └── api/bakery/      # 面包店业务接口
│   ├── web-antd/            # Ant Design 应用
│   ├── web-ele/             # Element Plus 应用
│   ├── web-naive/           # Naive UI 应用（主要业务）
│   │   └── src/
│   │       ├── api/bakery/  # 前端面包店 API
│   │       └── views/bakery/# 面包店业务页面
│   └── web-tdesign/         # TDesign 应用
├── packages/                # 模板核心包（已忽略）
├── internal/                # 内部工具配置（已忽略）
├── docs/                    # 模板文档（已忽略）
└── playground/              # 模板示例（已忽略）
```

## 已忽略的目录

以下目录已在 `.gitignore` 中配置，不会提交到团队仓库：

- `packages/` - vue-vben-admin 核心包
- `internal/` - 内部工具和配置
- `docs/` - 模板官方文档
- `playground/` - 模板演示示例

## 团队代码规范

### 提交代码

只提交团队自定义代码，主要包括：

- `apps/` 目录下的所有内容
- 项目根目录的配置文件（如 `.gitignore`, `package.json` 等）

### 更新模板

如需更新 vue-vben-admin 模板版本：

```bash
# 拉取官方最新代码
git fetch origin
git merge origin/master

# 解决可能的冲突
# 然后推送到团队仓库
git push bakery main
```

## 常见问题

### Q: 为什么不直接提交模板代码？

A: vue-vben-admin 是一个活跃的开源项目，定期会有更新。如果将模板代码提交到团队仓库，会导致：
1. 仓库体积过大
2. 难以跟踪团队自定义代码
3. 更新模板时冲突频繁

### Q: 如何确认哪些文件是团队自定义的？

A: 通过 `.gitignore` 文件可以清楚区分：
- `apps/` 目录下的文件都是团队自定义的业务代码
- `packages/`, `internal/`, `docs/`, `playground/` 是模板代码

### Q: 首次拉取项目后，缺少 packages 目录怎么办？

A: 这是正常的，只需要按照本指南从官方仓库拉取模板即可。
